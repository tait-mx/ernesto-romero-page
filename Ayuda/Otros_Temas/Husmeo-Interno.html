Husmeando alrededor de SuperCollider


Puedes inspeccionar mucha de la estructura interna de la librería de clases y de otras estructuras de datos.

Frecuentemente, esto es útil para fines de investigación y de depuración.


Definiciones de Clase, Implementaciones y Referencias


Seleccionar el nombre de cualquier clase (e.g. Object) y pulsar Cmd-j abrirá su archivo de definición de clase.


** Nota: Cualquier pulsamiento de teclas que abra archivos indetifica los archivos utilizando las Cadenas de Caracteres de Supercollider, las cuales sólo pueden contener caracteres ASCII hasta el punto de código 127. Si alguna parte de la ruta de tu instalación SuperCollider contiene códigos de caracteres mayores que 127 (como la vieja convención de Mac para identificar carpetas de programas con Option-F), la apertura de archivos va a fallar automáticamente.


El seleccionar el nombre de cualquier método (e.g. play) y pulsar Cmd-y abrirá una ventana mostrando todas las implementaciones de ese método y sus argumentos. El seleccionar una de esas clases y métodos (e.g. Sample:play) y pulsar Cmd-j abrirá la definición de clase a la altura de ese método. (Tomar en cuenta que Cmd-y sólo muestra implementaciones y no indica herencia).


El seleccionar cualquier texto (e.g. SCWindow o asStream) y pulsar Shift-Cmd-y abrirá una ventana mostrando todas las referencias al texto seleccionado, e.g. cada lugar en el que es usado dentro de la librería de clase (esto no va a encontrar llamadas de métodos compilados con "byte-codes" especiales como "value").


SC tiene un navegador de Clases gráfico, el cual muestra todos los métodos, argumentos, subclases, variables de instancia y variables de clase. Utilizando los botones del navegador, puedes fácilmente desplazarte a la superclase de la clase, a las subclases, la fuente de la clase, la fuente de los métodos, el archivo de ayuda (si hay alguno), revisar referencias o implementaciones de métodos, e incluso abrir un navegador de red para ver la entrada correspondiente en el repositorio SVN en línea.

Husmeando en Clases


El archivo de ayuda Class documenta algunos de estos métodos de husmeo.


A pesar de que puedes accesar estas estructuras de datos, si guardas cosas en ellas, puedes romper algo.


Collection.dumpInterface; // imprime todos los métodos de instancia definidos para esta clase


Collection.class.dumpInterface; // imprime todos los métodos de clase definidos para esta clase

Collection.methods.collect(_.name);

// imprime todos los métodos de instancia a los que esta clase responde

// imprime todos los métodos de clase a los que esta clase responde

// en orden alfabético

// también muestra de qué clase el método es heredado

// no incluye los métodos de las clases Object o Class

// para métodos de clase, haz Meta_Collection.dumpMethodList

Husmeando en Métodos


Lo mismo sucede aquí, si guardas cosas en los Métodos, puedes romper algo.


Collection.findMethod('select'); // ¿tiene este métodos?


Array.findMethod('select'); // esta clase no


Array.findRespondingMethodFor('select'); // trepa el árbol de clases para encontrar el método


Collection.findMethod('select').dump; // encuentra un objeto Método


Collection.findMethod('select').argNames.dump; // imprime el nombre de sus argumentos


Collection.findMethod('select').varNames.dump; // imprime el nombre de sus variables locales


// imprime su código ( principalmente para depurar el compilador).

Collection.findMethod('select').dumpByteCodes;


Collection.dumpByteCodes('select'); // una versión más corta de lo anterior


{ 1 + 2 }.dump; // esto es una Función ("Function")


{ 1 + 2 }.def.dump; // obtén su FunctionDef


{ 1 + 2 }.def.dumpByteCodes; // imprime su código



Husmeando en Ventanas


(

// crea algunas ventanas en las que husmear

5.do({ arg i;

var w, b;

w = Window.new("husmear " ++ i.asString, 

Rect.new( 200 + 400.rand, 69 + 300.rand, 172, 90 ));

w.front;

b = Button.new( w, Rect.new( 23, 28, 127, 25 ));

b.states = [["BLAM-O", Color.red]];

}))


Window.allWindows.dump; // imprime una lista de todas las Windows ("Ventanas") abiertas


// un poco más útil, imprime sus nombres

Window.allWindows.collect({ arg w; w.name }).postln;


(

// cambia el color de fondo de todas las ventanas abiertas

Window.allWindows.do({ arg window;

window.view.background = Color.new(0.5 + 0.5.rand, 0.5 + 0.5.rand, 0.5 + 0.5.rand);

}))


Window.closeAll; // cierra todas las ventanas (esto cerrará las ventanas del servidor)

Husmeando en el Intérprete


Al evaluar texto en el intérprete, la variable 'this' siempre se refiere al intérprete.


this.dump; // muestra los valores de todas las variables a-z del intérprete 


this.clearAll; // define a todas las variables a-z como nil


g = this.compile("(1 + 2).postln"); // compila algún texto en una Función


g.postln; // ves, g es una Función


g.value; // evalúa g


this.interpret("(1 + 2).postln"); // interpreta algún texto


this.interpretPrint("1 + 2"); // interpreta algún texto e imprime el resultado
