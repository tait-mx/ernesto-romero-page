Routine (Rutina)


Superclase: Thread


Las Rutinas son funciones que pueden devolver (entregar un valor) a la mitad y después continuar donde 

se quedaron cuando sean llamadas otra vez. Las Rutinas pueden ser usadas para implementar co-rutinas 

como se les encuentra en Scheme y otros lenguajes.

Las Rutinas sun útiles para escribir cosas que se comporten como Streams (flujos).

Las Rutinas heredan comportamientos para operaciones matemáticas y filtrado de Stream.


*new(func, stackSize, seed) 


Crea una instancia de Routine con una funci'on dada (func).

El stackSize (tamaño de la Pila de datos) y el random seed 

(semilla de aleatoriedad) pueden ser remplazados si se desea.


(

a = Routine.new({ 1.yield; 2.yield; });

a.next.postln;

a.next.postln;

a.next.postln;

)



value(inval)         // valor(valor de entrada)

resume(inval)    //  resumir(valor de entrada) - continuar desde donde se quedó

next(inval)   // siguiente(valor de entrada)


Todos estos son sinónimos del mismo método.


La función de la Routine es iniciada si no ha sido llamada aún o es 

resumida desde donde se quedó. El argumento inval es pasado como el argumento

para la función de la Routine si esta siendo iniciada o como el resultado del método yield

si está siendo resumida desde un yield. El resultado del método será

lo que la Routine produzca.


Básicamente existen dos condiciones para una Routine: una es cuando la rutina inicia. El otro caso

es que la rutina continúa despues de que se ha aplicado el método yield.


Cuando la rutina inicia (al llamar los métodos de arriba), se está pasado el primer inval.

Este inval es accesible como el argumento de la función de la rutina.


(

Routine { arg inval;

inval.postln;

}.value("hola rutina");

)


Cuando hay un yield en la rutina, la siguiente vez que se llame next (o un sinónimo), la rutina continúa 

desde ahí, y se tiene la oportunidad de ingresar un dato desde afuera. Para acceder a ese valor desde

la rutina que continúa, hay que asignar el resultado de la llamada yield a una variable:


(

r = Routine { arg inval;

var valorEntregadoPorYield;

inval.postln;

valorEntregadoPorYield = 123.yield;

valorEntregadoPorYield.postln;

}

)


r.value("hola rutina");

r.value("adios mundo");



El caso típico es que el nombre inval (o inevent) sea reusado, en vez de declarar una variable como "valorEntregadoPorYield":


(

r = Routine { arg inval;

inval.postln;

inval = 123.yield;

inval.postln;

}

)


r.value("hola rutina");

r.value("adios mundo");



Comúnmente una rutina usa múltiples yields, en donde el valor de inval es reasignado repetidamente.


(

r = Routine { arg inval;

inval.postln;

5.do { arg i;

inval = (i + 10).yield;

inval.postln;

}

}

)

(

5.do {

r.value("hola rutina").postln;

}

)




reset (reiniciar)


Hace que la rutina reinicie desde el principio la siguiente vez que sea llamada.

Una Routine no se puede aplicar un reset ella misma excepto si se llama con un

método yieldAndReset.


Ver también en la clase Object :

yield(outval)  //  producir(valor de salida)

yieldAndReset(outval) // producirYReiniciar(valor de salida)

alwaysYield(outval) // siempreProducir(valor de salida)


Si la función de una Routine regresa entonces siempre producirá nil hasta que se reinicie.





play(clock, quant)

clock: un Clock (reloj), TempoClock por defecto

quant: ver el archivo de ayuda de Quant   


En la aplicación SuperCollider, una Routine puede ser ejecutada usando un Clock, al igual que lo puede hacer cualquier Stream.

Cada vez que una Routine hace un yield, debe de hacerlo con un número de punto flotante (float - decimal), el reloj interpretará eso, 

usualmente haciendo una pausa por esa cantidad de segundos, y después resumirá la rutina, entregandole el tiempo actual del reloj.



usando Object-idle: 


idle(time)   //  ausente(tiempo)

dentro de una rutina, devuelve valores hasta que este tiempo transcurra. El tiempo es medido en relación al reloj del proceso (the thread's clock).

(ver Routine)

// durante 6 segundos, devuelve 200, después continúa

(

r = Routine {

199.yield;

189.yield;

200.idle(6);

199.yield;

189.yield;

};

fork {

loop {

r.value.postln;

1.wait;

}

}

);

// el valor también puede ser un stream o una función

(

r = Routine {

199.yield;

189.yield;

Routine { 100.do { |i| i.yield } }.idle(6);

199.yield;

189.yield;

};

fork {

loop {

r.value.postln;

1.wait;

}

}

);



Variables de instancia que se pueden acceder




Routine hereda de Thread,  quien permite tener acceso  a algunas de sus variables de instancia:


beats  // tiempos - como en múscica

devuelve los tiempos transcurridos (tiempo lógico) de la rutina. Los tiempos (beats) no proceden cuando la rutina 

no esta siendo ejecutada.

seconds // segundos

devuelve los segundos transcurridos (tiempo lógico) de la rutina. Los segundos no proceden cuando la rutina 

no esta siendo ejecutada.


clock  // reloj

devuelve el reloj del proceso (the thread's clock). Si no ha sido ejecutado, es el SystemClock (reloj del sistema)



(

r = Routine { arg inval;

loop {

// thisThread se refiere a la rutina

postf("beats: % segundos: % tiempo: % \n", 

thisThread.beats, thisThread.seconds, Main.elapsedTime

);

0.7.yield;

}

}.play;

)


r.stop;

r.beats;

r.seconds;

r.clock;





Ejemplo de Routine:


(

var r, outval;

r = Routine.new({ arg inval;

("->inval fue " ++ inval).postln;

inval = 1.yield;

("->inval fue " ++ inval).postln;

inval = 2.yield;

("->inval fue " ++ inval).postln;

inval = 99.yield;

});


outval = r.next('a');

("<-outval fue " ++ outval).postln;

outval = r.next('b');

("<-outval fue " ++ outval).postln;

r.reset; "reset (reinicio)".postln;

outval = r.next('c');

("<-outval fue " ++ outval).postln;

outval = r.next('d');

("<-outval fue " ++ outval).postln;

outval = r.next('e');

("<-outval fue " ++ outval).postln;

outval = r.next('f');

("<-outval fue " ++ outval).postln;

)


// wait (espera)


(

var r;

r = Routine {

10.do({ arg a;

a.postln;

// A veces se usa el método wait para pausar una rutina

// Esto hace que se espere por un segundo entre cada número

1.wait;

});

// Se espera medio segundo antes de decir "hecho"

0.5.wait;

"hecho".postln;

}.play;

)


// waitUntil  (esperar hasta que)


(

var r;

r = Routine {

var veces = { rrand(1.0, 10.0) }.dup(10) + thisThread.beats;

veces = veces.sort;

veces.do({ arg a;

waitUntil(a);

a.postln;

});

// Se espera medio segundo antes de decir "hecho"

0.5.wait;

"hecho".postln;

}.play;

)



// Usando Routine para definir los estados de un boton en marcha

(

var actualizacion, w, b;

w = SCWindow.new("Ventana de estados", Rect(150,SCWindow.screenBounds.height-140,380,60));

// una manera conveniente de definir la etiqueta del botón

actualizacion = { 

|but, string| but.states = [[string.asString, Color.black, Color.red]]; 

but.refresh;

};

b = SCButton(w, Rect(10,10,360,40));

b.font_(Font("Impact", 24));

actualizacion.value(b, "solo hay 1 estado");

// si una acción debería de hacer algo diferente cada vez que es llamada, una rutina es la

// la opción adecuada para usar. Esto es mejor que crear variables afuera y definirlas

// desde la función action para mantener el estado de una acción a la siguiente


b.action_(Routine { |boton|

rrand(15, 45).do { |i|

actualizacion.value(boton, "%. aun hay solo 1 estado".format(i + 2));

0.yield; // detenerse aqui

};

w.close;

});


w.front;

   )


// dibujando dinámicamente en una ventana con Pen

(

var w, cuanto = 0.02, string;


w = Window.new("mecer", Rect(100, 100, 300, 500)).front;

w.view.background_(Color.new255(153, 255, 102).vary);


string = "mecer ".dup(24).join;


w.drawHook = Routine {

var i = 0;

var size = 40;

var func = { |i, j| sin(i * 0.07 + (j * 0.0023) + 1.5pi) * cuanto + 1 };

var scale;

Pen.font = Font("Helvetica-Bold", 40);

loop {

i = i + 1;

string.do { |char, j|

scale = func.value(i, j).dup(6);

Pen.fillColor = Color.new255(0, 120, 120).vary;

Pen.matrix = scale * #[1, 0, 0, 1, 1, 0];

Pen.stringAtPoint(char.asString,

((size * (j % 9)) - 10) @ (size * (j div: 9))

);

};

0.yield // detenerse aqui, regresar algo sin importancia

}

};


fork { while { w.isClosed.not } { defer { w.refresh }; 0.04.wait; } };


w.front;


)
