Function (Función)


superclase: AbstractFunction




Una Function es una referencia a una [FunctionDef] y a su [Frame] de contexto que lo define. Cuando una FunctionDef es encontrada en tu código es metida en la pila de funciones como una Función. Una Function puede ser evaluada usando el método 'value'. Ver el archivo de ayuda Funciones para una introducción básica.


Como heredan de [AbstractFunction], las Functions pueden responder a operaciones matemáticas al crear una nueva Function. Por ejemplo:


(

var a, b, c;

a = { [100, 200, 300].choose }; // una Function

b = { 10.rand + 1 }; // otra Function

c = a + b; // c es una Function.

c.value.postln; // evalúa c e imprime el resultado

)


Ver [AbstractFunction] para ejemplos de composicion de funciones.




Acceso



def


Obtiene la definición de la FunctionDef de la Function



Evaluación


value(...args)


Evalúa la FunctionDef a la que se refiere la Function. Los argumentos dados son entregados a la Function.


{ arg a, b; (a * b).postln }.value(3, 10);


valueArray(..args..)



Evalúa la FunctionDef a la que se refiere la Function. Si el último argumento es un Array o List, entonces es desempacado y agregado a los otros argumentos (si existen) de la Function. Si el último argumento no es un Array o List entonces es lo mismo que con el método 'value'


{ arg a, b, c; ((a * b) + c).postln }.valueArray([3, 10, 7]);


{ arg a, b, c, d; [a, b, c, d].postln }.valueArray([1, 2, 3]);


{ arg a, b, c, d; [a, b, c, d].postln }.valueArray(9, [1, 2, 3]);


{ arg a, b, c, d; [a, b, c, d].postln }.valueArray(9, 10, [1, 2, 3]);


valueEnvir(...args)


Como value arriba. Los nombres de argumentos no provistos son buscados en el Environment actual.


(

Environment.use({

~a = 3;

~b = 10;

{ arg a, b; (a * b).postln }.valueEnvir;

});

)



valueArrayEnvir(..args..)


Evalúa la FunctionDef a la que se refiere la Function. Si el último argumento es un Array o una Lista entonces es desempacado y añadido a los otros argumentos (si hay) de la Function. Si el último argumento no es un Array o una Lista pasa lo mismo que con el método 'value'. Los nombres de argumentos no provistos son buscados en el Environment actual.


loop


Repite esta función. Útil con Task y Clocks.


t = Task({ { "Estoy loopeado".postln; 1.wait;}.loop });

t.start;

t.stop;



defer(delta)


Retrasa la evaluación de esta Function por delta segundos. Usa AppClock. 


{ "2 segundos han pasado.".postln; }.defer(2);



dup(n)


Devuelve un Array que consiste en los resultados de n evaluaciones de esta Function.


x = { 4.rand; }.dup(4);

x.postln;



! n


equivalente a dup(n)


x = { 4.rand } ! 4;

x.postln;



sum(n)


devuelve la suma de n valores producidos.


{ 4.rand }.sum(8);



bench(print)


Devuelve la cantidad de tiempo que esta función se toma para evaluar.  print es un booleano indicando si el resultado es impreso. El valor por defecto es true.


{ 1000000.do({ 1.0.rand }); }.bench;



fork(clock, quant, stackSize)


Devuelve una Routine (Rutina) usando al receptor como su función, y la ejecuta en un TempoClock.


{ 4.do({ "Procesando en paralelo...".postln; 1.wait;}) }.fork;



block


Interrupción de un bucle. Llama al receptor con un argumento que es una función que regresa del método block. Para salir del bucle, llamar .value en la función entregada. Se puede pasar un valor a esta función y este valor será devuelto desde el método block.



block {|break|

100.do {|i|

i.postln;

if (i == 7) { break.value(999) }

};

}


thunk


Regresa unThunk, que es un valor no evaluado que puede ser usado en un cálculos.


x = thunk { 4.rand };

x.value;

x.value;



flop


Devuelve una función que, al ser evaluada con argumentos anidados, hace una expansión multicanal al evaluar la función receptora para cada canal.


f = { |a, b| if(a > 0) { a + b } { -inf } }.flop;

f.value([-1, 2, 1, -3.0], [10, 1000]);

f.value(2, 3);




envirFlop


Como flop, pero implementa un argumento de entorno (environment argument) pasando  (valueEnvir). 

Menos eficiente en generación que flop, pero no es muy diferente en la evaluacón.


f = { |a| if(a > 0) { a + 1 } { -inf } }.envirFlop;

e = (a: [20, 40]);

e.use { f.value }



case(cases)


Function implementa un método case que permite  evaluaciones condicionales con múltiples casos. Como el receptor representa el primer caso esto puede se escrito simplemente como pares de funciones de prueba con sus correspondientes funciones a ser evaluadas en caso de que sean ciertas (true). A diferencia de Object-switch,  este es puesto en línea y es por eso igual de eficiente que usar condicionanles if anidados.


(

var i, x, z;

z = [0, 1, 1.1, 1.3, 1.5, 2];

i = z.choose;

x = case

{ i == 1 }   { \no }

{ i == 1.1 } { \mal }

{ i == 1.3 } { \mal }

{ i == 1.5 } { \mal }

{ i == 2 }   { \mal }

{ i == 0 }   { \cierto };

[i,x].postln;

)



Manejo de excepciones



Para los siguientes dos métodos un return ^ dentro del mismo receptor no puede ser atrapado. Returns en métodos llamados por el receptor están bien.



try(manejador)


Ejecuta al receptor. Si una excepción es arrojada la función manejador que la atrapa es ejecutada con el error como argumento. El manejador mismo puede re-arrojar un error si se desea.


protect(manejador)


Ejecuta al receptor. La función manejador que limpia es ejecutada con un error como argumento, o nil si no hay error. El error continúa teniendo efecto.


Ejemplos:


// sin manejador de excepciones

value { 8.zorg; \no_continuo.postln; }


try { 8.zorg } {|error| error.postln; \limpia.postln; }; \continuo.postln;


protect { 8.zorg } {|error| error.postln; }; \no_continuo.postln;


try { 123.postln; 456.throw; 789.postln } {|error| [\atrapa, error].postln };


try { 123.postln; 789.postln } {|error| [\atrapa, error].postln };


try { 123.postln; nil.throw; 789.postln } {|error| [\atrapa, error].postln };



protect { 123.postln; 456.throw; 789.postln } {|error| [\alSalir, error].postln };


protect { 123.postln; 789.postln } {|error| [\alSalir, error].postln };


(

try {

protect { 123.postln; 456.throw; 789.postln } {|error| [\alSalir, error].postln };

} {|error| [\atrapa, error].postln };

)


value { 123.postln; 456.throw; 789.postln }


value { 123.postln; Error("que paso?").throw; 789.postln }


(

a = [\aaa, \bbb, \ccc, \ddd];

a[1].postln;

a[\x].postln;

a[2].postln;

)


(

try {

a = [\aaa, \bbb, \ccc, \ddd];

a[1].postln;

a[\x].postln;

a[2].postln;

} {|error| \atrapado.postln; error.dump }

)


(

try {

a = [\aaa, \bbb, \ccc, \ddd];

a[1].postln;

a[\x].postln;

a[2].postln;

} {|error| \atrapado.postln; error.dump; error.throw }

)


(

protect {

a = [\aaa, \bbb, \ccc, \ddd];

a[1].postln;

a[\x].postln;

a[2].postln;

} {|error| \atrapado.postln; error.dump }

)



Audio


play(target, outbus, fadetime, addAction)


Esta es probablemente la manera más simple de obtener audio del SC3. Envuelve la Function en un SynthDef (añadiendo un ugen Out si es necesario), crea e inicia un nuevo Synth con él, y devuelve el objeto Synth. Un Linen es agregado también para evitar clicks, el cual esta configurado para permitir que el Synth resultante pueda cambiar su argumento \gate (puerta) o pueda responder a un mensaje de liberación.


target (blanco, objetivo) - un Node, Server o Nil. Un Server será convertido en el grupo por defecto de ese servidor. Nil será convertido en en el grupo por defecto del Server por defecto.

outbus (conducto de salida) - el bus de salida para reproducir el audio. Esto es equivalente a Out.ar(outbus, theoutput). El valor por defecto es 0.

fadeTime (timepo de desvanecimiento) - el tiempo de fadein (desvanecimiento de entrada). El tiempo por defecto es 0.02 segundos, lo cual es justo lo necesario para evitar un click. Este será también el tiempo de fadeout (desvanecimiento de salida).

addAction (acción de agregado) - ver Synth para una lista de addActions válidos. Por defecto se usa \addToHead (agregar a la cabeza).


x = { arg frec = 440; SinOsc.ar(frec, 0, 0.3) }.play; // esto devuelve un objeto Synth;

x.set(\frec, 880); // notar que se puede cambiar el argumento frec

x.defName; // el nombre del SynthDef resultante (derivado del valor hash de la Function. Ver hash en la clase Object)

x.release(4); // fadeout de 4 segundos


Muchos de los ejemplos en SC3  hacen uso de la sintaxis Function.play. Notar que la reutilización de un código como este en un SynthDef requiere agregar un ugen Out.


// las siguientes dos líneas producen resultados equivalentes

{ SinOsc.ar(440, 0, 0.3) }.play(fadeTime: 0.0); 

SynthDef("yuda-FuncPlay", { Out.ar(0, SinOsc.ar(440, 0, 0.3))}).play;


Function.play es a veces más conveniente que SynthDef.play , en particular para ejemplos  y  pruebas rápidas. El último tiene algunas opciones adicionales, como tiempos de retraso (lagtimes) para controles, etc. Cuando la reutilización y la máxima flexibilidad son de gran importancia, SynthDef y sus métodos varios son usualmente la mejor opción.


scope(numChannels, outbus, fadeTime, bufsize, zoom)


Como play arriba, pero reproduce en el internal Server (servidos interno), y llama a Server-scope a abrir una ventana visualizadora en la que se ve la salida. Actualmente solo funciona en OSX.

numChannels - El número de canales que se mostrarán en la ventana visualizadora, comenzando desde outbus. El valor por defecto es 2.

outbus (conducto de salida) - el bus de salida para reproducir el audio. Esto es equivalente a Out.ar(outbus, theoutput). El valor por defecto es 0. 

fadeTime (timepo de desvanecimiento) - el tiempo de fadein (desvanecimiento de entrada). El tiempo por defecto es 0.02 segundos, lo cual es justo lo necesario para evitar un click.

bufsize - El tamaño del buffer para el ScopeView. El valor por defecto es 4096

zoom - Un valor de acercamiento para el eje X del scope. Valores más grandes muestran más. El valor por defecto es 1.


{ FSinOsc.ar(440, 0, 0.3) }.scope(1)



plot(duration, server, bounds, minval, maxval, parent)


Calcula la duración en valores de segundo de la salida de esta función, y la grafica en una ventana GUI. A diferencia de play y scope no no trabaja con UGens Out, así que tu función devolverá un UGen o un Array de UGens. El plot será calculado en tiempo real.


duration - La duración de la función a plotear en segundos. El valor por default es 0.01.

server - El Server en el que se va a calcular el ploteado. Este debe estar corriendo en tu máquina local, pero no necesita ser el servidor interno. Si nil el servidor por defecto será usado.

bounds - Una instancia de Rect o Point indicando las delimitaciones de la ventana de ploteado.

minval - el valor mínimo del plot.  El valor por default es -1.0.

maxval - el valor máximo del plot.  El valor por default es 1.0.

parent (padre) - una ventana dentro de la cual colocar el plot. Si nil, una ventana será creada para ti.


{ SinOsc.ar(440) }.plot(0.01, bounds: GUI.window.screenBounds);

{ {|i| SinOsc.ar(1 + i)}.dup(7) }.plot(1);




Conversión




asSynthDef(rates, prependArgs, outClass, fadetime)


Devuelve un SynthDef basado en esta Function, agregando un Linen y un ugen Out si es necesario.


rates - Un Array de velocidades y tiempos de retraso (rates y lagtimes) para los argumentos de la función (ver SynthDef para más detalles).

outClass - La clase del ugen de salida como un símbolo. El usado por defecto es \Out. 

fadeTime - un tiempo de fadein (desvanecimiento de entrada). El valor por defecto es 0.




asDefName


Realiza asSynthDef (ver arriba), envía el def resultante al servidor local y devuelve el nombre del SynthDef. Esto es asíncrono.


x = { SinOsc.ar(440, 0, 0.3) }.asDefName; // esto debe completarse primero

y = Synth.new(x);


asRoutine


Devuelve una Routine (Rutina) usando esto como su argumento func.



inEnvir(envir)


Convierte una función en una función "environment-safe". Ver [Environment]  para más detalles.


// imprime nil porque ~a es leida de topEnvironment, no e

e = (a: "lo tengo", f: { { ~a.postln }.defer(0.5) });

e.use { e.f };


// imprime "lo tengo" porque { ~a.postln } esta ahora ligado al entorno e

e = (a: "lo tengo", f: { { ~a.postln }.inEnvir.defer(0.5) });

e.use { e.f };
