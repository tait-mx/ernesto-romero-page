InFeedback lee una señal de un bus con una marca de tiempo del ciclo actual o el anterior.



Hereda de: Object : AbstractFunction : UGen : MultiOutUGen : AbstractIn

*ar(bus, numChannels)

bus – el índice del bus del cual leer.

numChannels – el número de canales (p.ej. canales adyacentes) de los cuales leer. El valor por defecto es 1. No se puede modular este número asignándolo como argumento en una SynthDef.



Cuando los diferentes ugens de salida Out, OffsetOut, XOut) escriben datos a un bus, los mezclan con los del ciclo actual, pero sobreescriben todos los datos del ciclo anterior. (ReplaceOut sobreescribe todos los datos por igual.) Así, dependiendo del orden de los nodos y qué synths están escribiendo al bus, los datos en un bus dado pueden ser del ciclo actual o del anterior en el momento de la lectura. In.ar revisa la marca de tiempo de cualquier data que lee y asigna 0 a cualquier dato del ciclo anterior (para usar dentro del nodo, los datos permanecen el bus). Esto es suficiente para datos de audio, pues evita la retroalimentación, pero para datos de control, es útil poder leer datos desde cualquier lugar en el orden de nodos. Por esta razón, In.kr también lee los datos que son anteriores al ciclo actual.



En algunos casos también podemos leer datos de audio desde un nodo posterior en el orden actual de nodos. Este es el propósito de InFeedback. El retardo introducido por esto es el del tamaño del bloque, que equivale a aproximadamente 0.0014 sec en el tamaño por defecto de bloque (default block size) y la velocidad de muestreo. (Ver el archivo resonador para ver las implicaciones de esto.)



El comportamiento variable de mezcla y sobreescritura de los ugens de salida puede hacer que el orden de ejecución sea crucial. Por ejemplo, con un orden de nodos como el siguiente, el UGen InFeedback en el Synth 2 sólo recibirá datos del Synth 1

(-> = write out; <- = read in):



Synth 1 -> busA este synth sobreescribe la salida de Synth3 antes de que llegue a Synth2

Synth 2 (con InFeedback) <- busA 

Synth 3 -> busA



Si Synth 1 fuera movido después de Synth 2, entonces el InFeedback del Synth 2 recibiría una mezcla de la salida entre Synth 1 y Synth 3. Esto también sería cierto si Synth 2 estuviera después de Synth 1 y Synth 3. En ambos casos, los datos del Synth 1 y Synth 3 tendrían la misma marca de tiempo (ya sea del ciclo actual o anterior), así que nada sería sobreescrito.



Debido a esto, es útil alocar un bus separado para retroalimentación. Con la siguiente combinación el Synth 2 recibirá datos del Synth 3 sin importar la posición del Synth 1 en el orden de nodos.



Synth 1 -> busA

Synth 2 (con InFeedback) <- busB 

Synth 3 -> busB + busA 



El segundo ejemplo debajo demuestra este asunto.



Ver también LocalIn y LocalOut.





Ejemplos



modulación de retroalimentación de audio:



(

SynthDef(\help_InFeedback, { arg out=0, in=0;

var input, sound;

input = InFeedback.ar(in, 1);

sound = SinOsc.ar(input * 1300 + 300, 0, 0.4);

Out.ar(out, sound);



}).play;

)



esto muestra cómo un nodo puede leer audio de un bus que está siendo escrito por un synth posterior:



(

SynthDef(\help_InFeedback, { arg out=0, in=0;

Out.ar(out,

InFeedback.ar(in, 1)

);

}).send(s);

SynthDef(\help_SinOsc, { arg out=0, freq=440;

Out.ar(out, SinOsc.ar(freq, 0, 0.1))

}).send(s);

)

x = Bus.audio(s, 1);



// lee del bus n play al bus 0 (silencioso)

a = Synth(\help_InFeedback,[\in, x, \out, 0]);



// ahora reproduce un synth después de este, tocando en el bus x

b = Synth.after(a, \help_SinOsc, [\out, x]);

// añade otro synth antes de a que también escribe al bus x

// ahora no puedes oir b, pues sus datos son del ciclo anterior, y está siendo sobreescrito por c

c = Synth.before(a, \help_SinOsc, [\out, x, \freq, 800]);

// libera c y puedes oir a b de nuevo

c.free;

x.free;

a.free; b.free;



El ejemplo debajo implementa un resonador. Observa que deberás substraer el tamaño de bloque (blockSize) para que la afinación sea correcta. Ve LocalIn para un ejemplo equivalente.

(

var play, imp, initial;

SynthDef("testRes", {

play = InFeedback.ar(10, 1); // 10 es el canal del feedback

imp = Impulse.ar(1);

// feedback

OffsetOut.ar(10, DelayC.ar(imp + (play * 0.995), 1, 

440.reciprocal - ControlRate.ir.reciprocal)); // subtract block size

OffsetOut.ar(0, play);

}).play(s);

// Compara esto para afinación

{ SinOsc.ar(440, 0, 0.2) }.play(s, 1);
