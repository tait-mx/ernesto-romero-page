Mensajes 


Mandar mensajes es la manera de hacer las cosas en un lenguaje orientado a objetos. Un mensaje consiste en un selector de mensaje quien nombra el tipo de operación, un receptor a quien es enviado el mensaje y en algunos casos una lista de argumentos que dan información adicional a la operación. Un mensaje siempre devuelve un resultado. El tipo de resultado depende del tipo de mensaje. En el caso por defecto, el valor devuelto es el receptor mismo.


Los mensajes pueden ser escritos usando operadores binarios, notación funcional o notación de receptor.


Mensajes de operador binario


Un selector de operador binario es cualquier cadena de caracteres (string) de la lista de caracteres de operadores binarios legales:


! @ % & * - + = | < > ? /


Una excepción es que ningún operador puede comenzar con // o /* que son delimitadores de comentario.


Una expresión de operador binario consiste en dos expresiones con un operador binario entre ellas.


1 + 2 // la suma de uno y dos


a - b // la diferencia de a y b


x < 0.0 // responde si x es menor que cero


Un operador binario también puede ser un identificador seguido por  dos puntos.


10 rrand: 100


Jerarquía de operadores


No hay tal. Todas las operaciones binarias tienen la misma jerarquía y se asocian de izquierda a derecha.

Por ejemplo, la expresión:


a * b + c * d


es equivalente a:


((a * b) + c) * d


y no:


(a * b) + (c * d)

Por lo tanto es usualmente un mejor estilo poner paréntesis  en tus expresiones.


Mensajes en notación de función


El selector de mensaje precede la lista de argumentos entre paréntesis. El primer argumento de la lista de hecho es 

el receptor.


sin(x) // el seno de x

max(a, b) // el máximo entre a y b



Mensajes en notación de receptor


Una llamada de método en notación de función puede ser convertida a notación de receptor poniendo el receptor antes del nombre del método seguido de un punto como se muestra abajo.


max(a, b)    es equivalente a :   a.max(b)

sin(x)    es equivalente a :   x.sin

otro ejemplo:


g(f(a, b), c) 


es equivalente a :


g(a.f(b), c)


es equivalente a :


f(a, b).g(c)


es equivalente a :


a.f(b).g(c)



Valores por Defecto de Argumentos


Podrías llamar a una función o a un método con menos o mas argumentos de los que fue declarado para aceptar. Si se pasan menos argumentos, aquellos argumentos que no se pasan son enviados a un valor por defecto si alguno es dado en la definición de la función o del método, si no son enviados a nil. Si demasiados argumentos son pasados, el exceso de argumentos se coleccionan en un Array o son ignorados dependiendo de si la función o el método tienen un argumento de elipsis (explicado en Funciones). Cuando se llama a una función o a un método con cero argumentos se pueden omitir los paréntesis:


// It returns their sum. This syntax will be explained in the section on Functions.

// Regresa su suma. Esta sintáxis será explicada en la sección de Funciones.


x = { arg a=1, b=2; a + b }; 


z = x.value;    // el valor de z se fija en 3. (por defecto a es 1 y b es 2)


z = x.value(10); // el valor de z se fija en 12. (a es 10, y b por defecto es 2)


z = x.value(10, 5); // el valor de z se fija en 15. (a es 10, b es 5)


z = x.value(10, 5, 9); // el valor de z se fija en 15. (a es 10, b es 5, 9 es ignorado)



Argumentos de Palabra Clave


Los Argumentos para los Métodos pueden ser especificados con el nombre por el que fueron declarados en la definición del método. Estos argumentos son llamados argumentos de palabra clave (keyword). Cualquier argumento puede ser pasado como un argumento de palabra clave (keyword) a excepción del receptor 'this'. Los argumentos de palabra clave deben estar despues de cualquier argumento normal (conocidos como 'posicionales'), y pueden ser especificados en cualquier orden. Si una palabra clave es especificada y no hay un argumento que le corresponda entonces es ignorada y una advertencia será impresa. Esta advertencia puede ser apagada globalmente haciendo la siguiente llamada:


keywordWarnings(false)


Si un argumento de palabra clave y un argumento posicional especifican al mismo argumento, entonces el valor del argumento de palabra clave anula el valor del argumento posicional.


Por ejemplo el método de clase 'ar' de la clase SinOsc toma los argumentos llamados freq, phase, mul, y add en ese orden. Todas las siguientes son llamadas legales para ese método.


SinOsc.ar(800, pi, 0.2, 0); // todo los argumentos normales: freq, phase, mul, add


// freq = 800, mul = 0.2, los otros toman los valores por defecto.

SinOsc.ar(800, mul: 0.2); 


// freq = 800, phase = pi, mul = 0.2, add toma su valor por defecto 0.

SinOsc.ar(phase: pi, mul: 0.2, freq: 800);


// el valor 1200 de la palabra clave anula el valor 800 del argumento posicional

SinOsc.ar(800, freq: 1200); 


SinOsc.ar(zorg: 999); // la palabra clave inválida imprime una advertencia


Los argumentos de una Función pueden ser especificados también por argumentos de palabra clave cuando se usa el mensaje 'value'.


// Los argumentos de funciones pueden ser especificados por palabra clave

{ arg a=1, b=2, c=3; [a, b, c].postln }.value(b: 7, c: 8);


También se pueden usar argumentos de palabra clave cuando se usa el método 'perform'


SinOsc.perform('ar', phase: pi, mul: 0.2, freq: 800);


El costo de usar argumentos de palabra clave


Cuando se usan argumentos de palabra clave hay un costo en la ejecución por establecer la correspondencia del que deberías estar conciente. El costo puede valer la pena si la velocidad no es crítica.
