Bus representación de un bus en el servidor


La representación del lado del cliente de un bus de audio o de control en un servidor. Encapsula todos los mensajes OSC que un bus puede recibir. Maneja la asignación y la liberación de los índices de los canales, para que no tengas que preocuparte por conflictos. El número de buses de control, de audio, de entrada y de salida es fijo y no puede ser cambiado después de que el servidor haya sido arrancado. Para más información ver ClientVsServer y Server-Architecture.


Toma en cuenta que al usar la clase Bus para asignar un bus multicanal no se "crea" un bus multicanal, sino que simplemente se reserva una serie de índices de bus adyacentes mediante los asignadores de bus del objecto Server. unBus.index simplemente devuelve el primero de esos índices. Al usar un Bus con un UGen In o Out, no hay nada que te impida leer o escribir un rango más grande, o  codificar "en duro" a un bus que ha sido asignado. Eres responsable de asegurarte de que el número de canales corresponda y de que no haya conflictos.


Los objetos Bus no deben ser creados o modificados dentro de un SynthDef.


Nota: Los n índices de bus inferiores estás reservados para salidas y entradas de hardware, donde  n = server.options.numOutputBusChannels + server.options.numInputBusChannels.


Buses de salida de hardware: 0 .. (numOutputBusChannels - 1)

Buses de entrada de hardware: numOutputBusChannels .. (numOutputBusChannels + numInputBusChannels - 1)

Indice del primer bus privado: numOutputBusChannels + numInputBusChannels


No intentes utilizar los buses de entrada y de salida de hardware como buses privados.



Métodos de Clase


Bus.control(server, numChannels);

Asigna un bus de control en el servidor. 

Valores por defecto: servidor por defecto, 1 canal.

Bus.audio(server, numChannels);

Asigna un bus de audio en el servidor. 

Valores por defecto: servidor por defecto, 1 canal.

Bus.alloc(rate, server, numChannels);

Asigna un bus de cualquiera de las dos velocidades según sea especificado por los símbolos:

\control or \audio

Bus.new(rate, index, numChannels);

Este método no asigna un índice de bus, pero asume que tú ya

has asignado el índice de bus apropoiado y que puedes proporcionarlo.

Bus.newFrom(bus, offset, numChannels=1);

Este método crea un nuevo Bus que es un subconjunto del bus. El bus estará a la misma velocidad

que el bus de entrada.

offset es el índice dentro del bus dado. 

numChannels es el número deseado de canales.

Si la combinación de offset y de numChannels está fuera del "rango" del bus de entrada, aparecerá 

un mensaje de error. 


Métodos de Instancia


index - Obtiene el índice del bus. Normalmente no necesitarías hacer esto, ya que las instancias de Bus pueden ser enviadas directamente como entradas para UGens o argumentos para sintetizadores.

free - Devuelve los índices del bus al asignador de buses del servidor, para que puedan ser reasignados.

rate - Obtiene la velocidad del Bus. Esto es un símbolo, ya sea \control o \audio.

numChannels - Obtiene el número de canales del Bus.


server - Obtiene el objeto Server del Bus.


asMap - Devuelve un símbolo que consiste de la letra 'c' o 'a' (para control o audio) seguida del índice del bus. Esto puede ser usando al asignar las entradas de control (argumentos) de un nodo sintetizador al bus. Ver el archivo de ayuda de Node para más información acerca de la asignación controles a buses.


(

a = Bus.control(s, 1).set(440);

b = Bus.control(s, 1).set(0.01);

)

(

SynthDef(\rlpf, { |ffreq, rq|

Out.ar(0, RLPF.ar(WhiteNoise.ar(0.2), ffreq, rq))

}).play(s, [\ffreq, a.asMap, \rq, b.asMap]);

)


subBus(offset,numChannels) - Obtén un nuevo Bus que es un subgrupo de este bus (ver  *newFrom).


Métodos Asíncronos de los Buses de Control


Los siguientes comandos sólo aplican a los buses de control y son asíncronos. Para buses de control con acceso sincrónico, uno debe utilizar los buses control compartido del servidor interno y SharedIn / SharedOut.


value_(aFloat) - Asigna todos los canales a este valor de punto flotate. Este comando es asíncrono.

set(...values) - Una lista de valores para cada canal del bus de control. La lista de valores proporcionada no debe ser más grande que el número de canales. Este comando es asíncrono


setn(values) - Como set pero recibe un arreglo como argumento.


get(action) - Obtén el valor actual de este bus de control. Este comando es asíncrono. action es una función que será evaluada cuando el servidor responda, el valor actual del bus será enviado a la función como argumento. Para buses multicanal utilizar getN.


getn(count, action) - Obtén los valores actuales de este bus de control. Este comando es asíncrono. count es el número de canales a leer, empezando por el primer canal de este bus. action es una función que será evaluada cuando el servidor responda, los valores actuales del bus serán enviados como argumento.


Métodos de Paquetes OSC


getMsg - Devuelve un mensaje del tipo  /c_get para utilizar en paquetes OSC.


getnMsg(count) - Devuelve un mensaje del tipo  /c_getn para utilizar en paquetes OSC. count es el número de canales a leer, empezando por el primer canal de este bus. El valor por defecto es el número de canales de este bus.


setMsg(... values) -  Devuelve un mensaje del tipo  /c_set para utilizar en paquetes OSC.


setnMsg(values) -  Devuelve un mensaje del tipo  /c_setn para utilizar en paquetes OSC. values es un arreglo de valores a los que los canales adyacentes deben ser asignados, empezando por el primer canal de este bus.


fillMsg(value) -  Devuelve un mensaje del tipo  /c_setn para utilizar en paquetes OSC. value es el valor al que deben ser asignados los canales de este bus.

s.boot;

(

// algo con que jugar

SynthDef(\help_Bus, { arg out=0,ffreq=100;

var x;

x = RLPF.ar(LFPulse.ar(SinOsc.kr(0.2, 0, 10, 21), [0,0.1], 0.1),

ffreq, 0.1)

.clip2(0.4);

Out.ar(out, x);

}).send(s);


)


x = Synth(\help_Bus);


// obtén un bus

b = Bus.control(s);


// asigna la segunda entrada (ffreq) del sintetizador para que 

// lea de la salida del índice del bus

x.map(1, b);


// Al asignar el valor del bus mandas un mensaje /c_fill a cada canal

// del bus, asignado a éste al valor de punto flotante proporcionado

b.value = 100;

b.value = 1000;

b.value = 30;


// Ya que éste es un bus de un solo canal, esto tiene el mismo efecto

b.set(300);

b.numChannels.postln;


// multi-canal:  b.set(300,350);

// Obtén el valor actual. Esto es asíncrono, por lo que no puedes contar con que suceda

// inmediatamente

(

a = "esperando";

b.get({arg value; a = value; ("despues de que el servidor responda, a es asignada a:" + a).postln;});

("a es ahora:" + a).postln;

)


x.free;


b.free; // libéralo para que pueda ser reasignado!
