Cuando un servidor es arrancado, existe un grupo de nivel superior, con un ID de 0, el cual define la raíz del árbol de nodos. Este grupo está representado por la subclase RootNode (nodo raíz) de la clase Group. Si el servidor fue arrancado desde SClang (y no desde la línea de comandos), se creará un grupo por defecto (default_group) con un ID de 1. Este grupo es el grupo por defecto de todos los nodos al utilizar "estilo objetos" y es el grupo que obtendrás si proporcionas un servidor como objetivo ("target"). Si no especificas un objetivo o envías nil, obtendrás al grupo por defecto del servidor por defecto.


Server.default.boot;

a = Synth.new(\default); // crea un sintetizador en el grupo por defecto del servidor por defecto

a.group; // observa el ID del grupo


El grupo por defecto cumple una importante función: provee un árbol de nodos básico y predecible para que métodos como Server:scope y Server:record no tengan problemas de orden de ejecución. En el siguiente ejemplo, el nodo del osciloscopio vendrá después del grupo por defecto.


Server.internal.boot;


{ SinOsc.ar(mul: 0.2) }.scope(1);


// observa la ventana "post"

Server.internal.queryAllNodes; 


// nuestro sintetizador está dentro del grupo por defecto (ID 1).

// el nodo del osciloscopio viene después del grupo por defecto, 

// por lo que no hay ningún problema


Server.internal.quit;


Observa que el grupo por defecto es persistente: es creado en el método initTree del servidor (ejecutado junto con cualquier código guardado en la variable de instancia tree del servidor; ve Server para más detalles) y es recreado después rearrancar el servidor, después de presionar Cmd - ., y después de que todos los nodos son liberados. Al utilizar mensajes OSC, esto significa que un ID de nodo de 1 siempre puede ser utilizado:


s.sendMsg("/s_new", "snd", 1832,0,1); // añade en la cabeza del grupo 1


El grupo por defecto puede ser liberado específicamente, pero generalmente no hay ninguna razón por la cual uno quisiera hacer eso.


En general, uno debe añadir nodos al grupo por defecto en vez de al RootNode, a menos que uno tenga una razón específica para hacerlo (i.e. añadir alguna nueva funcionalidad, como grabar o visualizar en un oscilosopio, que dependa de un orden de nodos básico y predecible). Al utilizar "estilo objeto", el grupo por defecto es el objetivo por defecto para todos los nuevos nodos, por lo que normalmente los nodos no son añadidos al RootNode a menos que esto sea especificado explicitamente. Por supuesto que es posible hacerlo, pero sólo debe hacerse con un adecuado entendimiento de las cuestiones que involucra.


Por ejemplo, si uno colocara un sintetizador de "efectos" después del grupo por defecto y llamara a Server:record o a Server:scope, los nodos resultantes no estarían ordenados correctamente:


default group [

sintetizador fuente 1

sintetizador fuente 2

]

sintetizador grabador

sintetizador de efectos


En el ejemplo anterior, puede ser que el sintetizador que lleva a cabo la grabación no capture la señal de salida del sintetizador de efectos, ya que se encuentra después sen el orden de nodos.


En este caso, un mejor método sería crear un grupo dentro del grupo por defecto y colocar al sintetizador de efectos después de él.


default group [

source group [

sintetizador fuente 1

sintetizador fuente 2

]

sintetizador de efectos

]

sintetizador grabador
