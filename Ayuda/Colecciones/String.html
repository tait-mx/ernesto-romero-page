String arreglo de caracteres



Hereda de: Object : Collection : SequenceableCollection : ArrayedCollection : RawArray



String representa un arreglo de caracteres.

Los Strings pueden ser escritos literalmente utilizando comillas:



"my string".class.postln;



Métodos de clase



*readNew(file)



Lee los contenidos enteros de un Archivo y los devuelve como un String nuevo.



Métodos de Instancia



at(index)



Los Strings responden a .at de manera similar que otras colecciones con índices. Cada elemento es un Char.



"ABCDEFG".at(2).postln;



compare(aString)



Devuelve -1, 0 ó 1, dependiendo si el receptor debe ser clasificado antes que el argumento, o si debe ser clasificado después del argumento. Es un caso de comparación sensitiva.



< aString



Devuelve una variable Boolean si el receptor debe ser clasificado antes que el argumento

== aString



Devuelve una variable Boolean si los dos Strings son iguales.



post



Escribe el String en la ventana de post.



postln



Escribe el String y un salto de línea en la ventana de post.



postc postclnTal como post y postln arriba, pero formateado como comentario.





"This is a comment.".postcln;





postf



Escribe un String formateado con argumentos a la ventana de post. El caracter % en el String formateado es reemplazado por una representación en String de un argumento. Para escribir el caracter % utiliza \\% .



postf("this % a %. pi = %, list = %\n", "is", "test", pi.round(1e-4), (1..4))



this is a test. pi = 3.1416, list = [ 1, 2, 3, 4 ]



format



Devuelve una string formateada con argumentos. El caracter % en el String formateado es reemplazado por una representación en String de un argumento. Para escribir el caracter % utiliza \\% .



format("this % a %. pi = %, list = %\n", "is", "test", pi.round(1e-4), (1..4))



this is a test. pi = 3.1416, list = [ 1, 2, 3, 4 ]



matchRegexp(string, start, end)



comparación regular de expresiones POSIX. 

Devuelve 'true' si el receptor (un patrón de expresiones regular) es igual a un String pasado a él.

start es un desfazamiento a partir del cual empezar la búsqueda en el string. (Por defecto, 1). end indica donde detenerse.



"c".matchRegexp("abcdefg", 2, 5); // true

"c".matchRegexp("abcdefg", 4, 5); // false



"behaviou?r".matchRegexp("behavior"); // true

"behaviou?r".matchRegexp("behaviour"); // true

"behaviou?r".matchRegexp("behavir"); // false

"b.h.v.r".matchRegexp("behavor"); // true

"b.h.vi*r".matchRegexp("behaviiiiir"); // true

"(a|u)nd".matchRegexp("und"); // true

"(a|u)nd".matchRegexp("and"); // true

"[a-c]nd".matchRegexp("ind"); // false

"[a-c]nd".matchRegexp("bnd"); // true



findRegexp(string)



Búsqueda regular de expresiones POSIX. 



"foobar".findRegexp("o*bar");



"32424 334 /**aaaaaa*/".findRegexp("/\\*\\*a*\\*/");



"foobar".findRegexp("(o*)(bar)");



"aaaabaaa".findAllRegexp("a+");

asInteger



Devuelve un valor entero derivado de un String. Los string que comiencen con un valor no numérico devuelven 0.



"4".asInteger.postln;



asFloat



Devuelve un Float derivado del String. Los Strings que comienzan con valores no numéricos devuelven 0.



"4.3".asFloat.postln;



asSecs



Devuelve un Float basado en convertir un String de tiempo en formato (dd):hh:mm:ss.s.

Este es el método inverso a SimpleNumber:asTimeString. 

 

(45296.asTimeString).asSecs;

"32.1".asSecs;

"62.1".asSecs; // warns

"0:0:59.9".asSecs;

"1:1:1.1".asSecs;

"-1".asSecs; // neg sign supported

"-12:34:56".asSecs;

"12:-34:56".asSecs; // warns

"-23:12.3456".asSecs; //

"-1:00:00:00".asSecs; // days too.



catArgs(... args)



Concatena este String con los siguientes argumentos.



"These are some args: ".catArgs(\fish, SinOsc.ar, {4 + 3}).postln;



scatArgs(... args)



Igual a catArgs, pero con espacios en medio.



"These are some args: ".scatArgs(\fish, SinOsc.ar, {4 + 3}).postln;



ccatArgs(... args)



Igual a catArgs, pero con comas en medio.



"a String".ccatArgs(\fish, SinOsc.ar, {4 + 3}).postln;



catList(list) scatList(list) ccatList(list)



Igual a catArgs, scatArgs y ccatArgs arriba, pero toma una colección (usualmente una Lista o un Arreglo) como un argumento.



"a String".ccatList([\fish, SinOsc.ar, {4 + 3}]).postln;



split(separator)



Devuelve un Arreglo de Strings dividido en el separador. El separador es un Char, y no está incluído en el arreglo de salida. El separador default es $/, útil para rutas Unix.



"This/could/be/a/Unix/path".split.postln;

"These are several words".split($ ).postln;



ascii



Devuelve un Arreglo de números asci a partir de los caracteres del String.



"wertvoll".ascii;





find(string, ignoreCase, offset)



Devuelve el índice del string en el receptor, o nil si no se encuentra uno. Si ignoreCase es 'true', find no distingue entre mayúsculas y minúsculas. offset es el punto a partir del cual inicia la búsqueda en el string.



"These are several words".find("are").postln;

"These are several words".find("fish").postln;



findBackwards(string, ignoreCase, offset)



Igual a find, pero inicia al final del string.



// comparaa

"These words are several words".find("words"); // 6

"These words are several words".findBackwards("words"); // 24



findAll(string,  ignoreCase, offset)



Devuelve los índices del string en el receptor, o nil si no los encuentra.



"These are several words which are fish".findAll("are").postln;

"These are several words which are fish".findAll("fish").postln;



contains(string)



Devuelve una variable Boolean indicando si el String contiene la el argumento.



"These are several words".contains("are").postln;

"These are several words".contains("fish").postln;



containsi(string)



Igual a contains, pero sin atender a mayúsculas o minúsculas.



"These are several words".containsi("ArE").postln;



containsStringAt(index, string)



Devuelve una variable Boolean indicando si el String contiene el argumento en el índice especificado.



"These are several words".containsStringAt(6, "are").postln;



icontainsStringAt(index, string)



Igual a containsStringAt, pero sin atender a mayúsculas y minúsculas.



escapeChar(charToEscape)



Añade el caracter escape en la posición elegida.



"This will become a Unix friendly string".escapeChar($ ).postln;



tr(from, to)



Transliteración. Reemplaza todas las instancias de 'from' a 'to'.



":-(:-(:-(".tr($(, $)); //turn the frowns upside down



replace(find, replace)



Tal como tr, pero con strings como argumentos.



"Here are several words which are fish".replace("are", "were");



printOn(stream)



Escribe un String en un stream.



"Print this on Post".printOn(Post);



// equivalente a :

Post << "Print this on Post";

Igual a printOn, pero formateado como asCompileStream.



"Store this on Post".storeOn(Post);

Unix Support



Donde es relevante, el directorio actual de trabajo es el mismo donde se encuentra la aplicación de SuperCollider, y la shell es la shell Bourne (sh). Observa que cwd, y la shell en sí misma, no es persistente:



"echo $0".unixCmd; // print the shell (sh)

"pwd".unixCmd;

"cd Help/".unixCmd;

"pwd".unixCmd;



"export FISH=mackerel".unixCmd;

"echo $FISH".unixCmd;



Es posible, sin embargo, ejecutar comandos complejos:

"pwd; cd Help/; pwd".unixCmd;

"export FISH=mackerel; echo $FISH".unixCmd;



Si necesitaras que una variable de entorno persista, puedes utilizar setenv (ver debajo).



unixCmd(action, postOutput)



Ejecuta un String en la terminal utilizando la Bourne shell (sh). Devuelve el pid (nombre de un proceso) del proceso nuevo creado (utiliza Integer.pidRunning para probar si un pid está corriendo). Action es una Función que es llamada cuando el proceso es terminado. Recibe dos argumentos: el código de salida y el pid del proceso terminado. PostOutput es una variable Boolean que controla si la información del proceso es escrita en la ventana de post.



"ls Help".unixCmd;



"echo one; sleep 1; echo two; sleep 1".unixCmd { |res, pid| [\done, res, pid].postln };



unixCmdGetStdOut



Similar a unixCmd, excepto que el stdout del proeso es devuelto (de manera sincrónica), en vez de enviado a la ventana de post.



~listing = "ls Help".unixCmdGetStdOut; // Grab

~listing.reverse.as(Array).stutter.join.postln; // Mangle



runInTerminal



Ejecuta un String en una terminal nueva. (El string es incorporado en un archivo de script y ejecutado utilizando una shell. “usr/bash” es la shell usada por defecto pero tú puedes especificar qué shell utilizar como un argumento.)



"echo ---------Hello delightful SuperCollider user----------".runInTerminal;



setenv(value)



Le da a la variable de ambiente indicada en el String el valor del string value. Este valor persistirá hasta que sea cambiado o hasta que SC sea cerrado. Observa que si value es una ruta, tal vez tengas que utilizar el método standardizePath en él (ver abajo).



// todas las synthdefs en este directorio serán

// serán cargadas al arrancar el servidor local





"SC_SYNTHDEF_PATH".setenv("~/scwork/".standardizePath); 

"echo $SC_SYNTHDEF_PATH".unixCmd;



getenv



Devuelve el valor contenido en la variable de ambiente indicada por String.



"USER".getenv;



pathMatch



Devuelve un Arreglo que contiene todas las rutas iguales a este String. Se pueden usar comodines, sin recursión.



Post << "Help/*".pathMatch;



loadPaths



Realiza pathMatch (ver abajo) en este String, luego carga y ejecuta todas las rutas en el Arreglo resultante.



"Help/Collections/loadPaths example.rtf".loadPaths; //This file posts some text



load



Carga y ejecuta el archivo en la ruta representada por el receptor.



loadRelative



Carga y ejecuta el archivo en la ruta representada por el receptor, interpretando la ruta con relación al documento o archivo de texto en uso. Requiere que el archivo haya sido guardado.



resolveRelative



Convierte el receptor de una ruta relativa a una ruta absoluta, en relación al documento o archivo de texto en uso. Requiere que el archivo haya sido guardado.



standardizePath



Expande ~ a tu directorio home, y resuelve ligas simbólicas. Ver PathName para usos más complejos.



"~/".standardizePath; //This will print your home directory

Drawing Support



Los siguientes métodos deberán ser llamados dentro de un Window-drawHook o una función SCUserView-drawFunc, y sólo será visible una vez que la ventana o la vista (view) sea actualizada (refreshed). Cada llamada a Window-refresh SCUserView-refresh 'sobreescribirá' cualquier dibujo previo al ejecutar la función actualmente definida.



Ver también: Window, SCUserView, Color, and Pen.





Observa: para GUIs multiplataforma, utiliza en vez de esto Pen.stringAtPoint, Pen.stringInRect, Pen.stringCenteredIn, Pen.stringLeftJustIn, Pen.stringRightJustIn.





draw

Dibuja el String en el punto (Point) 0@0. Si no ha habido transformaciones de estado de los gráficos esta será la esquina superior izquierda de la ventana. Ver también Pen.



(

w = Window.new.front;

w.view.background_(Color.white);

w.drawHook = {

"abababababa\n\n\n".scramble.draw

};

w.refresh

)



drawAtPoint(point, font, color)



Dibuja el String en el punto (Point) utilizando la Font y Color especificados.



(

w = Window.new.front;

w.view.background_(Color.white);

w.drawHook = {

"abababababa\n\n\n".scramble.drawAtPoint(

100@100, 

Font("Courier", 30), 

Color.blue(0.3, 0.5))

};

w.refresh

)

(
