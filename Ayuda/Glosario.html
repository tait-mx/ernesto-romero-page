Glosario



buffer - Un encabezado y arreglo de valores de muestras de valores de punto flotante de 32 bits que residen en el servidor. Los buffers son usados como archivos de audio, líneas de delay, arreglos de controles globales, y arreglos de puntos de conexiones entre Synths. Son representados por la clase Buffer del lado del cliente.

class - clase - Una descripción del estado y comportamiento de un grupo de objetos.

client - cliente - SC está dividido en dos aplicaciones separadas: El cliente y el servidor. El cliente es donde el lenguaje de supercollider está implementado, y donde el usuario ejecuta código. El servidor realiza la síntesis del audio, contiene el árbol de nodos de síntesis y otros grupos y responde a mensajes Open Sound Control de parte del cliente. Ver ClientVsServer para más información.

group - grupo - Una "linked list" (lista enlazada) de nodos. Los grupos proporcionan maneras de controlar la ejecución de muchos nodos simultáneamente. Un grupo es una especie de nodo. Coloquialmente, se puede comprender un grupo como un agrupamiento ordenado de otros nodos, que puede incluir tanto Synths como otros grupos. Está representado por la clase Group del lado del cliente.

interface - interfase - El grupo de mensajes a los cuales un objeto responde.

instance - instancia - Uno de los objetos descritos por una clase.

instance variable - variable de instancia - Una parte del estado interno de un objeto.

message - mensaje - Una solicitud a un objeto para que realice una operación.

method - método - Una descripción de las operaciones necesarias para implementar un mensaje para una clase en particular.

MIDI - Un protocolo para mandar datos de control musical entre sintetizadores.

node - nodo - Un punto en el árbol de nodos ejecutado en orden "depth first traversal" (primero en profundidad: desde la raíz se lee cada "rama" hasta llegar al último nodo) por la maquinaria de síntesis. Hay dos tipos de nodos, síntesis y grupos. Éstos son representados por las clases del lado del servidor Synth y Group, y su superclase abstracta Node. El árbol de nodos define el orden de ejecución de los sintetizadores.

objeto - Algo que tiene datos representando el estado del objeto, y una serie de operaciones que pueden ser realizadas sobre él.

Open Sound Control - un protocolo definido por CNMAT en UCBerkeley para controlar sintetizadores. Ver http://opensoundcontrol.org/. SuperCollider se comunica entre el cliente y el servidor utilizando mensajes OSC sobre una red UDP o TCP.

OSC - Ver Open Sound Control.

polymorphism - polimorfismo - La abilidad que tienen diferentes tipos de objetos de responder de maneras distintas al mismo mensaje.

protocol - protocolo - Un grupo de mensajes que implementan una clase específica de comportamiento.

receiver - receptor - El objeto al cual se le envía un mensaje.

servidor - SC está dividido en dos aplicaciones separadas: El cliente y el servidor. El cliente es donde el lenguaje de supercollider está implementado, y donde el usuario ejecuta código. El servidor realiza la síntesis del audio, contiene el árbol de nodos de síntesis y otros grupos y responde a mensajes Open Sound Control de parte del cliente. Ver ClientVsServer para más información.

synth - sintetizador - Un módulo de procesamiento de sonido, basado en una definición de sintetizador particular. Similar a "voice" (voz) en otros sístemas. Se hace referencia a los sintetizadores utilizando un número. Está representado por la clase Synth del lado del cliente.

synth definition - definición de síntesis - Una definición para crear nuevos sintetizadores. Las definiciones de síntesis son como un patrón o un diseño para sintetizadores. Similar a un "instrumento" en otros sistemas. Representado por la clase SynthDef del lado del cliente.

TCP - Un protocolo para transmitir datos a través de una red.

UDP - Un protocolo para enviar datagramas a través de una red.
