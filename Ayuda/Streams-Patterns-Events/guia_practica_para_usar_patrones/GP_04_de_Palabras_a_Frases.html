De palabras a frases


Un patrón sencillo por sí mismo no es muy interesante. Pero los patrones se usan conjuntamente para obtener resultados más complejos. 


Patrones dentro de patrones de lista


Hemos visto que los patrones de listas (Pseq, Prand, etc.) retornan números a partir de una lista predeterminada, ya sea en algún orden dado o rearregladas de manera aleatoria. La lista también puede incluir otros patrones. Cuando un patrón de listas encuentra otro patrón en su lista, el patrón interno es incrustado al flujo. Es decir, el patrón interno asume el control del flujo hasta que se le acaben los valores que retorna. Entonces el control regresa al patrón exterior. Esto es como llamar a una función en medio de otra función. 


No hay un límite preestablecido para el número de niveles de incrustación. 


Si un patrón sencillo es como una palabra, un patrón de listas que usa otros patrones podría considerarse como una oración o frase. Se puede alternar entre distintos comportamientos ya sea de manera predecible, como en el ejemplo siguiente, o aleatoriamente usando algunos de los patrones de listas aleatorios. 


// Segmentos de escala, en la secuencia: arriba, arriba, abajo (repetir)

(

TempoClock.default.tempo = 1;

p = Pbind(

\degree, Pseq([

Pseries({ rrand(0, 7) }, 1, { rrand(4, 8) }), // arriba (paso = 1)

Pseries({ rrand(0, 7) }, 1, { rrand(4, 8) }), // arriba (paso = 1)

Pseries({ rrand(7, 14) }, -1, { rrand(4, 8) }) // abajo (paso = -1)

], inf),

\dur, 0.125

).play;

)


p.stop;



Pero se pone más divertido -- a los patrones de listas no les importa si están encerrando patrones de valores (como en el ejemplo anterior) o patrones de eventos. Esto significa que uno puede escribir un conjunto de patrones al estilo Pbind, cada uno representando una frase, e hilarlos conjuntamente. El siguiente ejemplo es más largo, pero solamente por el número mayor de patrones de frase. Sin embargo, la estructura es muy simple: Pxrand([Pbind(), Pmono(), Pmono()...], inf). Algunas frases están escritas con Pmono para deslizar entre notas. 


(

SynthDef(\bajo, { |freq = 440, gate = 1, amp = 0.5, slideTime = 0.17, ffreq = 1100, width = 0.15,

detune = 1.005, preamp = 4|

var sig,

env = Env.adsr(0.01, 0.3, 0.4, 0.1);

freq = Lag.kr(freq, slideTime);

sig = Mix(VarSaw.ar([freq, freq * detune], 0, width, preamp)).distort * amp

* EnvGen.kr(env, gate, doneAction: 2);

sig = LPF.ar(sig, ffreq);

Out.ar(0, sig ! 2)

}).memStore;

)


(

TempoClock.default.tempo = 132/60;

p = Pxrand([

Pbind( // notas repetidas

\instrument, \bajo,

\midinote, 36,

\dur, Pseq([0.75, 0.25, 0.25, 0.25, 0.5], 1),

\legato, Pseq([0.9, 0.3, 0.3, 0.3, 0.3], 1),

\amp, 0.5, \detune, 1.005

),

Pmono(\bajo, // salto de octava

\midinote, Pseq([36, 48, 36], 1),

\dur, Pseq([0.25, 0.25, 0.5], 1),

\amp, 0.5, \detune, 1.005

),

Pmono(\bajo, // salto de tritono

\midinote, Pseq([36, 42, 41, 33], 1),

\dur, Pseq([0.25, 0.25, 0.25, 0.75], 1),

\amp, 0.5, \detune, 1.005

),

Pmono(\bajo, // triada disminuida

\midinote, Pseq([36, 39, 36, 42], 1),

\dur, Pseq([0.25, 0.5, 0.25, 0.5], 1),

\amp, 0.5, \detune, 1.005

)

], inf).play(quant: 1);

)


p.stop;


Atajo de notación: Así como los arreglos se concatenan con ++, se pueden concatenar patrones de la misma manera. Escribir patrón1 ++ patrón2 es lo mismo que escribir Pseq([patrón1, patrón2], 1).



Algunas maneras de hilar patrones


Secuencialmente: Cada subpatrón sigue al anterior en el mismo orden cada vez. Usar Pseq o Pser.


Orden aleatorio: Los sub patrones en orden totalmente aleatorio (Prand), orden aleatorio sin repeticiones (Pxrand), u orden aleatorio de acuerdo a un conjunto de posibilidades (Pwrand). Pshuf crea un ordenamiento aleatorio y lo usa repetidamente. 


Indexado directo de arreglos: Patrones que se escogen en algún orden arbitrario mediante un índice. Esto da mayor control que Pwrand. Tanto Pindex como  Pswitch pueden ser usados para esto.


// segmentos de grados de escala, cada quinta elección es solamente impar (descendiendo)

(

var n = 10,

segmentosDeEscala = Array.fill(n, { |i|

if(i.odd) {

Pseries(11, -1, rrand(5, 10))

} {

Pseries(rrand(-4, 4), 1, i+2)

}

});


TempoClock.default.tempo = 1;

p = Pbind(

\degree, Pswitch(segmentosDeEscala, Pseq([Pwhite(0, n-1, 4), Pwhite(0, n-1, 1).select(_.odd)], inf)),

\dur, 0.125

).play;

)


p.stop;



Máquinas de estado finito (Pfsm, pdfsm)


Una máquina de estado finito es una manera de asociar un elemento con sus posibles sucesores. Es más cercano a una "gramática" que a una selección meramente aleatoria.  Pfsm define una máquina de estado finito como un conjunto de posibles "puntos de entrada", seguido por una lista de "estados" posibles de la máquina y, para cada estado, una lista de posibles estados que pueden seguir el estado actual. Los estados pueden ser valores sencillos o patrones, es decir, las frases pueden ser vinculadas a otras frases que "tengan sentido" en sucesión (y las transiciones no deseadas serán prevenidas). 


Si esto suena similar a una cadena de Markov, es debido a que la implementación de Pfsm es un caso especial de una cadena de Markov donde hay igual probabilidad de escoger el siguiente estado a partir de los sucesores válidos. En una cadena de Markov, las probabilidades son ponderadas de acuerdo al análisis de un flujo de datos. 


La página de ayuda de Pfsm incluye muy buenos ejemplos de cómo organizar valores sencillos y frases de patrones. También ver PG_Cookbook06_Phrase_Network para una aplicación de Pfsm para generar un sólo de jazz cursi. 


Pdfsm quiere decir "máquina de estado finito determinística", donde no hay selección aleatoria. 


Nota acerca de extensiones: Una buena implementación de cadenas de Markov para SuperCollider existe en el quark MathLib.



Librería de sub patrones nombrados


Una muy efectiva manera de administrar frases es hacer una biblioteca, o más precisamente, un Diccionario (Dictionary) de sub patrones y luego llamarlos uno a la vez. Psym es el patrón para hacer esto. La ventaja aquí es que uno puede guardar las frases en un lugar separado, mientras que el patrón que uno realmente reproduce es mucho más simple y describe la intención musical a un nivel más alto. 


// Usa el SynthDef bajo de arriba

(

~frases = (

repetido: Pbind(

\instrument, \bajo,

\midinote, 36,

\dur, Pseq([0.75, 0.25, 0.25, 0.25, 0.5], 1),

\legato, Pseq([0.9, 0.3, 0.3, 0.3, 0.3], 1),

\amp, 0.5, \detune, 1.005

),

octava: Pmono(\bajo,

\midinote, Pseq([36, 48, 36], 1),

\dur, Pseq([0.25, 0.25, 0.5], 1),

\amp, 0.5, \detune, 1.005

),

tritono: Pmono(\bajo,

\midinote, Pseq([36, 42, 41, 33], 1),

\dur, Pseq([0.25, 0.25, 0.25, 0.75], 1),

\amp, 0.5, \detune, 1.005

),

dim: Pmono(\bajo,

\midinote, Pseq([36, 39, 36, 42], 1),

\dur, Pseq([0.25, 0.5, 0.25, 0.5], 1),

\amp, 0.5, \detune, 1.005

)

);


TempoClock.default.tempo = 128/60;


// el patrón que controla a un nivel más alto es muy simple ahora

p = Psym(Pxrand(#[repetido, octava, tritono, dim], inf), ~frases).play;

)


p.stop;


Un patrón complicado con muchas incrustaciones puede ser difícil de leer ya que es más trabajo separar detalles al nivel de las notas que de la estructura global. El patrón que escoge las frases -- Pxrand(#[repetido, octava, tritono, dim], inf) --  se explica a sí mismo y Psym se encarga de los detalles transparentemente. 


Nota: Debido a que algunas manipulaciones especiales son necesarias para patrones de eventos, hay dos versiones de Psym. Psym se encarga de patrones de eventos, mientras que Pnsym es para patrones de valores. Se puede pensar de esta manera: Pbind puede ser contenido dentro de Psym, pero contiene Pnsym. 


( Psym ( Pbind ( Pnsym ) ) )


Bien:

Psym(**, (patrón1: Pbind(**))

Pbind(\algúnValor, Pnsym(**, (patrón1: Pwhite(**)))


Mal:

Pbind(\algúnValor, Psym(**, (patrón1: Pwhite(**)))



Cambiando entre patrones para valores individuales


En los anteriores ejemplos, si un patrón de lista encuentra a otro patrón como uno de sus valores de entrada, el sub patrón es incrustado en su totalidad antes de que se le permita continuar al patrón exterior. A veces uno quisiera obtener nada más un valor del sub patrón y luego escoger otro subpatrón para el siguiente evento. Pswitch, Psym y Pnsym tiene primos que hacen exactamente eso: Pswitch1, Psym1 y Pnsym1.


// alturas al azar en dos rangos distintos; se lanza una moneda al aire para decidir este evento 

// 70% abajo, 30% arriba

(

TempoClock.default.tempo = 1;

p = Pbind(

\degree, Pswitch1([Pwhite(7, 14, inf), Pwhite(-7, 0, inf)], Pfunc { 0.7.coin.binaryValue }),

\dur, 0.25

).play;

)


p.stop;


Comparar con el siguiente:


(

p = Pbind(

\degree, Pswitch([Pwhite(7, 14, inf), Pwhite(-7, 0, inf)], Pfunc { 0.7.coin.binaryValue }),

\dur, 0.25

).play;

)


p.stop;


Con Pswitch, uno de los elementos se escoge de la lista y sigue tocando hasta que se acaba. Pero la longitud de ambos patrones Pwhite es infinita, por lo que el primero que se escoja mantendrá el control. Pswitch1 hace el 'volado' (cara o cruz) cada evento e incrusta solo uno de ellos. 


Psym1 y  Pnsym1 se comportan de manera similar, escogiendo el nombre para buscar el patrón en cada evento. 


Tema relacionado: patrones condicionales


Pif  admite este tipo de estructura: si el siguiente valor de un patrón Booleano es verdadero, retorna el siguiente elemento del patrón A, de lo contrario lo toma del patrón B. Otra manera de escribir el ejemplo de Pswitch1 es usando pruebas booleanas directamente en los Pwhite en lugar de escribir una Pfunc para el tiro de la moneda. Esto puede ser más claro para leerse. Sin embargo, funciona solo si hay dos alternativas. Pswitch1 y Psym1 admiten cualquier número de elecciones. 


(

TempoClock.default.tempo = 1;

p = Pbind(

// traducción: if(0.7.coin) { rrand(-7, 0) } { rrand(7, 14 }

\degree, Pif(Pwhite(0.0, 1.0, inf) < 0.7, Pwhite(-7, 0, inf), Pwhite(7, 14, inf)),   

\dur, 0.25

).play;

)


p.stop;


Veremos más adelante en GP_06e_Control del Lenguaje que Pif puede ser usado en valores previamente calculados en el Pbind. Añada considerablemente a la inteligencia de la que Pbind es capaz, cuando sus flujos de valores están conscientes de otros valores en el evento.
