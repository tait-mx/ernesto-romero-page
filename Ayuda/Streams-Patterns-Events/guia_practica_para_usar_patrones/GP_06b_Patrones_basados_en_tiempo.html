Patrones basados en tiempo


Los "patrones basados en tiempo" son definidos aquí como patrones que usan tiempo como parte de sus cálculos. Naturalmente los patrones de eventos son impulsados por tiempo cuando son tocados por un reloj (y técnicamente se pueden requerir eventos de un flujo de eventos sin que sean reproducidos por un EventStreamPlayer -- ReproductorDeFlujosDeEventos --, pero esto no es un uso típico). 


La mayoría de estos patrones funcionan acordándose del tiempo actual del reloj en el momento en el que se incrusta el patrón al flujo de valores. El valor temporal usado para los cálculos es entonces el tiempo del reloj al tiempo de la evaluación menos el tiempo de comienzo -- es decir, el número de pulsos que han transcurrido desde que el patrón comenzó su incrustación. Se el patrón es incrustado varias veces, el tiempo de comienzo es también restablecido para que el patrón comience desde el principio. 


No hay nada que prevenga usar estos patrones fuera de un contexto de programación de itinerarios. En estos documentos ese contexto sería el de un patrón de eventos reproducido por un reloj, pero los flujos hechos a partir de estos patrones pueden ser usados en rutinas planificadas o también en funciones. Sin embargo, sólo un contexto de programación de itinerarios puede asegurar una coordinación precisa para la solicitud de valores. 


  • Ptime(repeticiones): Retorna la cantidad de tiempo transcurrido desde que se incrustó. Un uso típico de este patrón es detener un flujo/patrón de valores después de cierta cantidad de tiempo. 

Este patrón Pif usa Ptime para obtener valores de la bifurcación cierta para exactamente 4 pulso después de que el primer valor es solicitado. Después de eso, la condición será falsa y Pif revertirá a la bifurcación falsa, que es nula. Esto causa la detención del flujo. (Esto es como  Pfindur para patrones de eventos, pero Pif/Ptime funciona también con patrones de valores). 


// Este es un truco muy útil: como Pfindur pero con patrones de valores

(

p = Pbind(

\degree, Pif(Ptime(inf) < 4.0, Pwhite(-4, 11, inf)),

\dur, 0.25

).play;

)


  • Pstep(niveles, durs, repeticiones): Repite un valor 'nivel' durante su correspondiente duración y luego se mueve al siguiente valor. 
  • Pseg(niveles, durs, curvas, repeticiones): Similar a Pstep, pero interpola al siguiente valor en lugar de dar el paso de manera abrupta al final de la duración. La interpolación es lineal por defecto, pero cualquier curva de envolvente puede ser usada. 'niveles' y 'durs' deben ser patrones. 


// la curva es 5 - aquí se puede ver la curva, primero ascendiendo y luego descendiendo

Env(#[0, 1, 0], #[1, 1], 5).plot;


(

p = Pbind(

// usando \note, Pseg dará números de nota fraccionales

// no se puede usar \degree porque maneja números no enteros diferentemente

\note, Pseg(

Pwhite(-7, 19, inf), // números de nota cromáticos

// versión alternativa para números diatónicos

// PdegreeToKey hace la misma conversión que \degree --> \note

// PdegreeToKey(Pwhite(-4, 11, inf), Pkey(\scale), 12),

Pwhite(1, 4, inf) * 0.5,

5, inf),

\dur, 0.125

).play;

)


p.stop;



Usando envolventes como patrones


Env admite el protocolo de flujos: 'asStream' ('comoFlujo') convierte un Env en un flujo y valores coordinados en el tiempo pueden ser obtenidos de él usando 'next' ('siguiente'). El flujo retorna el valor que tendría el envolvente al momento del tiempo transcurrido. De la misma manera .at(tiempo) retorna el valor del envolvente en el tiempo especificado. 


e = Env.linen(1, 1, 1);

e.at(2); // == 1

e.at(2.5); // == 0.5


// imprime los valores del envolvente

r = fork {

var stream = e.asStream;

12.do({

stream.next.postln;

0.25.wait;

});

};


// Usar un envolvente para panear notas de izquierda a derecha y de vuelta

p = Pbind(

\degree, Pwhite(-4, 11, 32),

\pan, Env(#[-1, 1, -1], #[2, 2], \sin),

\dur, 0.125

).play;


p.stop;


Los parámetros del envolvente nodoDeLiberación y el nodoDeBucle no tienen efecto ya que sólo tienen sentido si son usados dentro de un Sinte con un EnvGen con compuerta. 


Cuando acaba el envolvente, el flujo mantendrá el valor final indefinidamente. El truco Pif(Ptime(inf) < totalTime, Env(...)) sirve para hacer que se detenga.


// Usar un envolvente para panear notas de izquierda a derecha y de vuelta

// Toca sólo un ciclo

(

p = Pbind(

// cambiar a inf: no necesitamos saber exactamente cuántos eventos necesitamos

\degree, Pwhite(-4, 11, inf),

\pan, Pif(Ptime(inf) <= 4.0, Env(#[-1, 1, -1], #[2, 2], \sin)),

\dur, 0.125

).play;

)


p.stop;


// Para mantener el envolvente iterando en un bucle, envolver Pif dentro de Pn

(

p = Pbind(

\degree, Pwhite(-4, 11, inf),

\pan, Pn(Pif(Ptime(inf) <= 4.0, Env(#[-1, 1, -1], #[2, 2], \sin)), inf),

\dur, 0.125

).play;

)


p.stop;
