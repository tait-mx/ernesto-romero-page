Métodos de control del lenguaje


Algunos patrones imitan métodos de control en el estilo del lenguaje: condicionales (Pif), bucles (Pwhile) y limpieza de errores (Pprotect).


  • Pif(condición, cierto, falso, predeterminado): Evalúa el patrón 'condición' que retorna cierto o falso. Luego, un valor es tomado de la bifurcación cierto o falso antes de volver a evaluar la condición de nuevo. El valor o patrón 'predeterminado' entra en juego cuando las bifurcaciones 'cierto' y 'falso' han dejado de producir valores (es decir, retornan nulo). Si el valor predeterminado no es dado, Pif retorna control al patrón progenitor si hay un nulo en cualquier bifurcación. 


p = Pbind(

\degree, Pwhite(0, 11, inf),

// grados de escala nones tienen un valor rítmico más corto

\dur, Pif(Pkey(\degree).odd, 0.25, 0.5)

).play;


p.stop;



  • Pseed(semillaAleatoria, patrón): Los generadores de números aleatorios dependen de valores de "semilla" ("seed"); asignar una semilla específica produce un flujo repetible de números pseudo aleatorios. Pseed asigna la semilla aleatoria antes de incrustar 'patrón', con lo cual efectivamente reinicia el generador de números aleatorios al principio del patrón.


p = Pbind(

// la semilla aleatoria es generada una vez, cuando se crea el objeto del patrón

// de tal manera que la misma semilla se usa cada vez que el patrón reproduce

\degree, Pseed(0x7FFFFFFF.rand, Pseries({ rrand(-7, 0) }, Pwhite(1, 3, inf), { rrand(4, 10) })),

\dur, 0.25

);


q = p.play; // usa una semilla

q.stop;


r = p.play; // usa la misma semilla

r.stop;


// re-ejecutar p = Pbind... y la semilla será diferente



  • Pprotect(patrón, func): Igual que el método 'protect' para manejar errores, si un error ocurre mientras se obtiene el siguiente valor, la función será evaluada antes de que el error interrumpa la ejecución.
  • Ptrace(patrón, llave, flujoDeImpresión, prefijo): Para depuración, Ptrace imprime cada valor retornado. ¿El patrón realmente está haciendo lo que uno cree? Esto nos dirá. Un Ptrace es creado automáticamente por el mensaje 'trace': unPatrón.trace(llave, flujoDeImpresión, prefijo) --> Ptrace(unPatrón, llave, flujoDeImpresión, prefijo).
  • Pwhile(func, patrón): Igual que while: mientras la función evalúe cierta, el patrón es incrustado. La función es revisada una vez al principio y después, cuando el patrón ha llegado a su fin. Si se aplica a un patrón infinito no habrá bucle ya que el patrón no devolverá el control. 


// Pwhile y Ptrace

(

~go = true;

p = Pwhile({ ~go }, Pbind(

\degree, Pseries({ rrand(-7, 0) }, Pwhite(1, 3, inf), { rrand(4, 10) })

.trace(prefix: "degree: "),

\dur, 0.25

)).play;

)


~go = false; // detendrá el patrón completo cuando el Pbind llegue al final
