Patrones de repetición y restricción


Estos son esencialmente patrones de control de flujo. Cada uno toma un patrón fuente y ya sea que repita valores de él o se detenga temprano basado en una restricción predeterminada. 


Patrones de repetición


Estos patrones permiten la repetición de valores sencillos o (en el caso de Pn) patrones enteros.


  • Pclutch(patrón, conectado): Si el patrón 'conectado' es verdadero (true), Pclutch retorna el siguiente valore de 'patrón'. Si 'conectado' es falso, el valor previo del patrón es repetido. Es como el clutch de un coche: cuando está activado, el patrón se mueve adelante; cuando está desconectado, se mantiene donde está. 
  • Pn(patrón, repeticiones): Incrusta el patrón fuente un número 'repeticiones' de veces. También funciona con valores individuales: Pn(1, 5) produce el número 1 cinco veces. 
  • Pstutter(n, patrón): 'n' y 'patrón' son ambos patrones. Cada valor de 'patrón' es repetido 'n' veces. Si 'n' es un patrón, cada valor puede ser repetido un número diferente de veces. 
  • PdurStutter(n, patrón): Como Pstutter, excepto que el patrón de valores es dividido por el número de repeticiones (de tal manera que el tiempo total para un ciclo repetido es el valor de duración proveniente del patrón fuente).


Ver también Pstep, descrita en GP_06b_Patrones basados en Tiempo. Pstep se puede usar como Pstutter, pero la repetición es controlada por tiempo en lugar de repeticiones de cada elemento. 


// toca notas repetidos con diferente valor rítmico cada nueva altura

// usando Pstutter

p = Pbind(

// haciendo 'n' un flujo separado para que dur y degree lo compartan

\n, Pwhite(3, 10, inf),

\degree, Pstutter(Pkey(\n), Pwhite(-4, 11, inf)),

\dur, Pstutter(Pkey(\n), Pwhite(0.1, 0.4, inf)),

\legato, 0.3

).play;


p.stop;



// usando Pfin / Pn

// Pn del Pbind un bucle infinito

// Plazy construye un nuevo Pbind para cada iteración

// Pfin corta de tajo el Pbind cuando es tiempo de un valor nuevo


p = Pn(

Plazy {

Pbind(

\degree, Pfin(rrand(3, 10), rrand(-4, 11)),

\dur, rrand(0.1, 0.4) 

)

},

inf

).play;


p.stop;



// usando Pclutch

// la regla es: cuando un grado cambia, la duración también debe cambiar

// si Pdiff retorna 0, el grado no ha cambiado, entonces,

// los valores no cero del Pdiff "conectan" el clutch y permiten la generación de una nueva dur 

// de lo contrario, la vieja dur se mantiene

p = Pbind(

\degree, Pstutter(Pwhite(3, 10, inf), Pwhite(-4, 11, inf)),

\dur, Pclutch(Pwhite(0.1, 0.4, inf), Pdiff(Pkey(\degree)).abs > 0),

\legato, 0.3

).play;


p.stop;



Patrones de restricción (o interrupción)


En lugar de prolongar un flujo por repetición, estos patrones usan diferentes métodos para detener dinámicamente un flujo. Son especialmente útiles para modularizar la construcción de patrones. Una sección de nuestro código puede ser responsable de la generación de patrones numéricos o de eventos. Mediante los patrones de restricción, esa parte del código no tiene que saber cuántos eventos o qué tanto tiempo tocar, pueden simplemente retornar patrones infinitos que serán envueltos, en otra parte del código, por estos patrones, que determinarán según condiciones apropiadas su detención.


  • Pfin(cuenta, patrón): Retorna exactamente 'cuenta' valores del patrón fuente y luego se detiene. 
  • Pconst(suma, patrón, tolerancia): Produce números hasta que la suma alcance un número predefinido. El último valor producido es ajustado para dar exactamente la suma.
  • Pfindur(dur, patrón, tolerancia): Como Pconst, pero aplicando el comportamiento de restricción a los valores rítmicos del patrón. El patrón fuente corre hasta la duración especificada y luego se detiene. Es muy útil si uno sabe cuánto debe durar un comportamiento musical, pero no sabe cuántos eventos son necesarios para cubrir ese periodo de tiempo. 

// Dos variantes de la misma cosa

// Usar Pconst o Pfindur para crear segmentos de 4 pulsos con ritmo aleatorio

// Pconst y Pfindur pueden ambos asegurar que el ritmo total no sea mayor a 4.0


p = Pn(Pbind(

// siempre un do grave en el tiempo fuerte

\degree, Pseq([-7, Pwhite(0, 11, inf)], 1),

\dur, Pconst(4, Pwhite(1, 4, inf) * 0.25)

), inf).play;


p.stop;


p = Pn(Pfindur(4, Pbind(

\degree, Pseq([-7, Pwhite(0, 11, inf)], 1),

\dur, Pwhite(1, 4, inf) * 0.25

)), inf).play;


p.stop;



  • Psync(patrón, cuant, maxdur, tolerancia): Como Pfindur, pero no tiene que tener una duración fija. En su lugar, toca ya sea hasta que alcance maxdur (en caso del cual se comporta igual que Pfindur, ajustando el último evento para que la duración total alcance maxdur), o el patrón se detiene antes y el último evento es redondeado hasta el siguiente múltiplo entero de cuant. 


(

// en este caso, el patrón se detiene al alcanzar maxdur

// tiempo transcurrido = 4

var startTime;

p = (Psync(Pbind(

\dur, 0.25, // duración total = infinita

\time, Pfunc { startTime = startTime ? (thisThread.clock.beats.debug("tiempo")) }

), 1, 4) ++ Pfuncn({

thisThread.clock.beats.debug("Tiempo de finalizacion");

(thisThread.clock.beats - startTime).debug("Transcurrido");

nil

}, 1)).play;

)


(

// en este caso, el patrón se detiene antes de maxdur (4)

// la duración del Pbind (1.25) se redondea hasta 2 (el siguiente múltiplo de 1)

var startTime;

p = (Psync(Pbind(

\dur, Pn(0.25, 5), // duración total = 0.25 * 5 = 1.25

\time, Pfunc { startTime = startTime ? (thisThread.clock.beats.debug("time")) }

), 1, 4) ++ Pfuncn({

thisThread.clock.beats.debug("finish time");

(thisThread.clock.beats - startTime).debug("Transcurrido");

nil

}, 1)).play;

)
