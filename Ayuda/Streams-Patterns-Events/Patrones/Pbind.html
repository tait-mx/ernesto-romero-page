Pbind combina varios patrones de valores en el flujo de un evento al enlazar claves con palabras.

Inherits from: Object : AbstractFunction : Pattern


Pbind combina varios flujos de valores en el flujo de un evento. Cada flujo de valores es asignado a una o mas claves (keys) en el flujo del evento que resulta. Especifica un flujo de Events (eventos) en términos de diferentes patrones que están atados a diferentes claves en el Event.

Los patrones atados a claves se referencían como patrones (patterns) de valores y el Pbind mismo es nombrado un patron de evento (event patern)



Las claves usadas en un Pbind están determinadas usualmente por el mecanismo por defecto de Event y los controles están definidos por el SynthDef que se ejecutará. ( Ver SynthDef y Event abajo para una breve discusión sobre ambos en relación con Pbind)


Creación  / Métodos de Clase


*new (key1, pattern1, key2, pattern2 ...) // *nuevo(clave1, patrón1, clave2, patrón2 ...)

Los argumentos para Pbind son una serie alternante de claves y patrones.

Un patrón puede estar atado a un arreglo de claves. En este caso el patrón debe especificar

una secuencia cuyos elementos son arreglos con al menos la misma cantidad de elementos que

la cantidad de claves que hay.

SynthDef y Event


Las claves usadas en un Pbind están determinadas por el SynthDef usado y por la estructura del extensivo mecanismo por defecto provisto por Event. Esta sección provee una breve revisión de ambos.


Un SynthDef asigna un nombre a una interconexión de unidades generadoras para ser ejecutado como un synth en un sevidor. También asigna nombres de control (control names) a las entradas de control del synth. En el ejemplo siguiente el SynthDef\test tiene las entradas de control out, freq, amp, nharms, pan, y gate. 


SynthDef(\test, { | out, freq = 440, amp = 0.1, nharms = 10, pan = 0, gate = 1 |

var audio = Blip.ar(freq, nharms, amp);

var env = Linen.kr(gate, doneAction: 2);

OffsetOut.ar(out, Pan2.ar(audio, pan, env) );

}).memStore;

El SynthDef necesita ser descargado al servidor en el que será ejecutado. Usa .memStore en vez de .send para asegurar que cualquier patrón que esté usando este SynthDef tenga información sobre las entradas de control disponibles (ver SynthDesc). Alternativamente, .store puede ser usado para guardar el SynthDef en el disco y agregar el SynthDesc a la librería.


Un Event es un Dictionary (Diccionario)  que especifica una acción que se hará en respuesta a play y un incremento de tiempo que se devolverá en respuesta a delta.


Por defecto, los Events ejecutan synths en un servidor. Tales eventos de nota usan las siguientes claves:


instrument (\default) El synthdef a ser ejecutado

variant (nil, opcional) La variante a utilizar. Una variante es un conjunto de valores por defecto para los argumentos de un sintetizador 

(ver el archivo de ayuda de SynthDef)

server (Server.default) El servidor que ejecuta el synth

group ( 1) El nuevo synth o el synth depués o antes del cual el nuevo synth es colocado

addAction (0) cómo es colocado el synth en relación a un objetivo especificado por group

0 -> en la cabeza del grupo

1 -> en la cola del grupo

2 -> antes del grupo (puede ser un Synth)

3 -> después del grupo (puede ser un Synth)

delta (función) El tiempo haste el siguiente evento  en una secuencia de eventos, generalmente especificado indirecamente a través de dur.


Cuando el Event es ejecutado, crea un comando OSC para ejecutar el synth. Utiliza el nombre asignado por instrument para seleccionar el SynthDef a ser ejecutado. Los nombres de control del SynthDef (situados en su SynthDesc) son buscados en el evento y los corresponientes valores son incluidos en el comando.


El ejecutar un Synth es la acción normal llevada a cabo por un Event. La estructura de evento por defecto define varos otros tipos de eventos que pueden realizar una amplia variedad de acciones de servidor. Ver el archivo de ayuda de Event para una lista de tipos de evento.


Existe un número de nombres convencionales típicamente usados para identificar los controles en un synth.

 

out índice del bus de salida

in índice del bus de salida (para filtros, moduladores, etc)

gate puerta de la envolvente (no es nivel) - debe de tomar 1.0 como valor por defecto, borra el synth cuando es liberado

trig puerta de la envolvente (no es nivel) - debe de tomar 1.0 como valor por defecto, no borra el synth cuando es liberado

pan posición del paneo

bufnum número de buffer (usado en synths que utilizan Playbuf, DiskIn, etc)

sustain duración del synth

amp amplitud del synth

freq altura base del synth


Event implementa un esquema de  especificación por capas para algunos de sus comandos. En la siguiente lista, el primer nombre , que está más a la izquierda, es el nombre del control real, los nombres que están mas abajo e indentados  son formas más abstractas de especificar el control.


delta // (delta) El tiempo hasta el siguiente evento. Determinado generalmente por:

dur // (duración) El tiempo hasta el siguiente evento en una secuencia de eventos.

stretch // (expansión) Escala los tiempos del evento (i.e. stretch == 2 => las duraciones son el doble de largas)


sustain // (sostenimiento) Las duraciones del synth, determinadas típicamente (en unidades de tiempo extendidas) por:

legato La razón de la duración del synth y la duración del evento


amp // (amplitud) la amplitud del synth (típicamente entre 0 y 1), usualmente determinada por:

db La amplitud en decibeles


detunedFreq // (frecuenciaDesafinada) la altura de un synth, determinada por:

freq + detune;  // (frecuencia+desafinación) freq está determinada por

(midinote + ctranspose).midicps * harmonic;  // (notamidi+transposiciónc).midiciclosporsegundo * armónico;  midinote está determinada por

(note + gtranspose + root)/stepsPerOctave * octave * 12; // (nota + transpocicióng + raiz)/pasosPorOctava * octava * 12; note está determinada por

(degree + mtranspose).degreeToKey(scale, stepsPerOctave) // (grado + transposiciónm).gradoATecla(escala, pasosPorOctava)

El método play


Aunque el método play está de hecho definido en la clase Pattern, es útil revisarlo aquí:


play (clock, protoEvent, quant) // ejecutar(reloj, protoEvento, cuantificador) devuelve un EventStreamPlayer 

clock El reloj que planifica el EventStramPlayer, por defecto usa TempoClock.default

Los patrones que cambien gráficas deben usar AppClock.

protoEvent El evento inicial modificado por Pbind, por defecto usa Event.default.

quant Un valor de cuantificación usado por el reloj.

Cuando es un número, el patrón empezará en el siguiente múltiplo par

de ese número. También puede ser Quant, quien especifica la cuantificación, la posición en el tiempo 

dentro de esa cuantificación,

y un timingOffset   (despazamiento de tiempo). Ver Quant para mas detalles.




Control en Tiempo Real con EventStreamPlayer (Ejecutador de Flujo de Evento)


EventStreamPlayer  provee control en tiempo real a través de mute, unmute, stop, play y reset. 


(

SynthDef(\cfstring1, { arg i_out, freq = 360, gate = 1, pan, amp=0.1;

var out, eg, fc, osc, a, b, w;

fc = LinExp.kr(LFNoise1.kr(Rand(0.25, 0.4)), -1, 1, 500, 2000);

osc = Mix.fill(8, {LFSaw.ar(freq * [Rand(0.99, 1.01), Rand(0.99, 1.01)], 0, amp) }).distort * 0.2;

eg = EnvGen.kr(Env.asr(1, 1, 1), gate, doneAction:2);

out = eg * RLPF.ar(osc, fc, 0.1);

#a, b = out;

Out.ar(i_out, Mix.ar(PanAz.ar(4, [a, b], [pan, pan+0.3])));

}).memStore;


e = Pbind(

\degree, Pseq((0..12), inf), 

\dur, 0.2, 

\instrument, \cfstring1

).play; // devuelve un EventStream

)


( // una sesión interactiva

e.stop

e.play

e.reset


e.mute; // sigue ejecutandose, pero remplaza las notas con silencios


e.unmute;


e.reset; // restaura el flujo.

e.reset; // restaura el flujo.

e.reset; // restaura el flujo.

e.reset; // restaura el flujo.


e.pause; // lo retomará desde donde se pausó.


e.play;


e.stop; // restaurará el flujo antes de retomarlo.


e.play;


)

Envío a efectos.

La asignación a procesadores de efectos poede ser lograda definiendo el argumento \out con

el bus de entrada del efecto deseado. El Synth de efecto también debe ser creado. Una manera 

de hacerlo es con Synth.new


(

// synthdef de efecto- juega con el tiempo del delay y el pbind. :-P

SynthDef(\pbindefx, { arg out, in, tiempo1=0.25, tiempo2=0.5; 

var audio, efx;

audio = In.ar([20, 21], 2);

efx=CombN.ar(audio, 0.5, [tiempo1, tiempo2], 10, 1, audio); 

Out.ar(out, efx);

}).memStore;

)


// crea el synt de efecto

a = Synth.after(1, \pbindefx);
