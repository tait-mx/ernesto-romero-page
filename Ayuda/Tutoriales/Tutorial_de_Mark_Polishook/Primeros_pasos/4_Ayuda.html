Ayuda



SuperCollider tiene un sistema de ayuda incluido. Para ver la página principal de ayuda, presiona cmd-d (sin seleccionar nada). En esa página, haz clic en los temas subrayados o presiona cmd-d después de seleccionar texto. Otro documento útil es More-On-Getting-Help (Más-información-Sobre-Cómo-Conseguir-Ayuda).



En general, hay archivos de ayuda para las clases (palabras con mayúscula, tales como SinOsc, Array, Nil, etc.). Selecciona el nombre de una clase y presiona Cmd-d. Un archivo de ayuda, si existe alguno, se abrirá.



////////////////////////////////////////////////////////////////////////////////////////////////////



Para ver todos los archivos de ayuda de SuperCollider


evalúa:


Help.all



////////////////////////////////////////////////////////////////////////////////////////////////////


Para ver la ayuda de todos los unit generators 


evalúa


Help("Help/UGens").dumpToDoc("SuperCollider UGens (Unit Generators)")


Cada línea de texto en el documento que aparece contiene una palabra con el sufijo ".help". Haz doble clic en cualquiera de las palabras para seleccionarlas y presiona Cmd-D para abrir el archivo de ayuda correspondiente. Omite el sufijo ".help" de la palabra cuando hagas doble clic.




////////////////////////////////////////////////////////////////////////////////////////////////////


Definiciones de clase, implementaciones de mensajes, y el comando Find


Para ver el código fuente de las definiciones de clase, selecciona el nombre de una clase y presiona cmd-j


Para ver cómo una clase o clases implementan un mensaje particular, selecciona el nombre del mensaje y presiona cmd-y.


Utiliza los comandos 'Find' y 'Find Next', disponibles en el menú 'Edit', para buscar texto del primer documento abierto


////////////////////////////////////////////////////////////////////////////////////////////////////


grep


Utiliza grep en la Terminal (en la carpeta Aplicaciones->Utilidades) para buscar todas las ocasiones en que una palabra o frase aparecen. Por ejemplo, para ver todos los documentos que utilizan la clase LFSaw, evalúa (en la Terminal)


grep -r LFSaw /Applications/SuperCollider_f


Ya que las líneas en la terminal terminan de acuerdo con el tamaño de la ventana y no debido a esquemas que aumenten la legibilidad, puede que sea más sencillo escribir los resultados de grep a un archivo, como en el siguiente ejemplo:


// cambia el nombre de la ruta (el argumento después del signo '>', según como sea apropiado:

grep -r LFSaw /Applicaciones/SuperCollider_f/ > /Usuarios/tuDirectorioDeUsuario/Escritorio/grep_results
