Los controles (usualmente) son argumentos


Use controles, que usualmente se definen como argumentos a una ugenGraphFunc (función de gráfica de  ugen), ya sea cuando es creada y/o después de que está corriendo. Suministrar valores predeterminados a los argumentos para que el código sea más legible y para protegerlo contra error de usuario (como olvidar suministrar un valor para el argumento).



(

// 3 argumentos (controles) con valores predeterminados

SynthDef(

"conControles", 

{ arg freq = 440, beatFreq = 1, mul = 0.22;

Out.ar(

0,

SinOsc.ar([freq, freq+beatFreq], 0, mul)

)

}).load(s);

)


// artículos en el arreglo son pasados a los controles en un Sinte cuando es creado. 

z = Synth("conControles", [\freq, 440, \beatFreq, 1, \mul, 0.1]);


// evaluar esta línea después de que el sinte esté corriendo para restaurar sus controles

z.set(\freq, 700, \beatFreq, 2, \mul, 0.2);



////////////////////////////////////////////////////////////////////////////////////////////////////


Escribir los nombres de los controles y los valores apropiados en el arreglo dado como argumento a un sinte. Los nombres de control pueden ser dados como símbolos (que son nombrés únicos dentro del sistema de SuperCollider).


Synth("conControles", [\freq, 440, \beatFreq, 0.5, \mul, 0.1]);


o como 'cuerdas' (un arreglo de caracteres)


Synth("conControles", ["freq", 440, "beatFreq", 0.5, "mul", 0.1]);


De cualquier manera, el patrón es


[ nombreDeControl, valor, nombreDeControl, valor, nombreDeControl, valor].


Ver los archivos para Symbol y String en el sistema de ayuda de SuperCollider.


////////////////////////////////////////////////////////////////////////////////////////////////////


Una tercera manera de pasar controles a un sinte: 


Synth("conControles", [0, 440, 1, 1, 2, 0.1]);


En este caso, el patrón es


[ índiceDeControl, valor, índiceDeControl, valor, índiceDeControl].


////////////////////////////////////////////////////////////////////////////////////////////////////



Para ajustar un control


Use el mensaje .set para cambiar el valor de un control mientras el sinte esté corriendo.


(

SynthDef("restauraMisControles", { arg freq = 440, mul = 0.22;

Out.ar(

0,

SinOsc.ar([freq, freq+1], 0, mul)

)

}).load(s);

)


~unSinte = Synth("restauraMisControles", [\freq, 440, \mul, 0.06]);

~unSinte.set(\freq, 600, \mul, 0.25);


////////////////////////////////////////////////////////////////////////////////////////////////////


Variables globales


En el ejemplo previo, el caracter '~' antes de unSinte define una variable global. Una ventaja de usar una variable global es que no es necesario haberla declarado explícitamente, como en


var aSynth; // variables sin el '~' DEBEN ser declaradas primero!!


Más precisamente, el caracter ~ pone una variable llamada unSinte dentro de una instancia de un objecto conocido como currentEnvironment ('ambienteActual). Para mayor información, ver el documento Environment en el sistema de ayuda de SuperCollider.


////////////////////////////////////////////////////////////////////////////////////////////////////


Tiempos de demora


Use un arreglo de tiempos de demora para exponer que tanto toma deslizarse suavemente de un valor de control a otro. Escriba los tiempos de demora en un arreglo y colóquelo en el synthdef después del ugenGraphFunc, como por ejemplo


(

SynthDef("controlConDemoras", { arg freq1 = 440, freq2 = 443, mul = 0.12;

Out.ar(

0,

SinOsc.ar([freq1, freq2], 0, mul)

)

}, [4, 5]).load(s);

)


~unSinte = Synth("controlConDemoras", [\freq1, 550, \freq2, 344, \mul, 0.1]);

~unSinte.set(\freq1, 600, \freq2, 701, \mul, 0.05);


////////////////////////////////////////////////////////////////////////////////////////////////////


Plantillas de SynthDefs


El arreglo de tiempos de demoras implica que la plantilla de synthdef con dos componentes (discutido en 3_SynthDefs)


// una plantilla para un synthdef con dos componentes

SynthDef(

"aSinte", // primer argumento es un nombre

{ ... soy una funciónDeGráficaDeUgen ... } // segundo argumento es una ugenGraphFunc

)


puede ser revisada para incluir tres componentes.


// una plantilla redefinida para un synthdef con un arreglo de tiempos de demora

// la definición de clase para el arreglo de tiempos de demora los llama 'velocidades' ('rates')

SynthDef(

"unSinte", // nombre

{ ... soy una funciónDeGráficaDeUgen ... }, // ugenGraphFunc

[ ... tiemposDeDemora ... ] // velocidades

)
