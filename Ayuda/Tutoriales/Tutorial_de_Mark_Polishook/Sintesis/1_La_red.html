Redes y el cliente/servidor


SuperCollider 3 utiliza un modelo de cliente/servidor para operar a través de una red. Esto significa que los usuarios escriben programas cliente que le indican al servidor que debe realizar una acción, es decir, solicitan servicio. Dichas solicitudes pueden ocurrir localmente dentro de un ordenador o pueden estar distribuidas de manera remota entre dos o más ordenadores. Ya sea que los ordenadores estén localizados en el mismo espacio o distribuidos en distintos puntos del mundo no significa ninguna diferencia mientras estén conectados en una red.


Los programas cliente dentro de Supercollider normalmente especificarán las definiciones de síntesis (cómo un sonido en particular será realizado) y la programación de la síntesis (en qué momento un sonido particular será ejecutado). El servidor (o servidores) sintetizará(n) el audio de acuerdo a las instrucciones del cliente (o clientes).


Como resumen, los clientes solicitan, los servidores responden.


////////////////////////////////////////////////////////////////////////////////////////////////////


Ejemplos de cliente/servidor:


// EJ. 1 - ejecuta cada línea, una cada vez:

// definir un proceso de síntesis y hacer una solicitud al servidor a través del cliente.

////////////////////////////////////////////////////////////////////////////////////////////////////


// define un servidor con un nombre y una dirección:

s = Server("aServer", NetAddr("127.0.0.1", 56789)); // "localhost" es un sínonimo para la dirección IP // "127.0.0.1"


// inicia el servidor:

s.boot;


// definir un sintetizador:

SynthDef("sine", { Out.ar(0, SinOsc.ar(440, 0, 0.2)) }).send(s);


// programar (ejecutar) la síntesis:

s.sendMsg("s_new", "sine", n = s.nextNodeID, 0, 1);


// detener el síntetizador (eliminarlo):

s.sendMsg("/n_free", n);


// (opcionalmente) detener el servidor

s.quit;


////////////////////////////////////////////////////////////////////////////////////////////////////

// EJ. 2

// Igual que el anterior, excepto que es realizado en 2 computadoras conectadas en red

////////////////////////////////////////////////////////////////////////////////////////////////////

// definir un servidor (remoto); representa un ordenador "en algún lugar" conectado al internet

// el número IP tiene que ser válido y el servidor, donde sea que esté, tiene que estar activo

// los servidores no pueden ser iniciados remotamente, p. ej., un programa en un ordenador no puede iniciar un servidor en otro

// este ejemplo asume que el servidor en el ordenador remoto fue iniciado desde supercollider y no desde la terminal

s = Server("aServer", NetAddr("... un número ip ...", 56789));


// define un sintetizador... exactamente como en el ejemplo anterior:

SynthDef("sine", { Out.ar(0, SinOsc.ar(440, 0, 0.2)) }).send(s);


// programar la síntesis ... exactamente como en el ejemplo anterior:

s.sendMsg("s_new", "sine", n = s.nextNodeID, 0, 1);


// detener el sintetizador (eliminarlo):

s.sendMsg("/n_free", n);

// EJ. 3:

// cliente/servidor en un ordenador vs. cliente/servidor en dos ordenadores

// el ejemplo anterior sin comentarios

// son idénticos, excepto que el ejemplo ejecutado en un ordenador explícitamente inicia el servidor

Servidores localhost e interno


El ejemplo anterior define objetos de servidor. Pero, generalmente, esto no es necesario ya que SuperCollider define dos objetos tales, el servidor localhost y los servidores internos al inicio. Son representados por ventanas en la parte inferior de la pantalla. Cada una de las ventanas tiene un botón de inicio para iniciar el servidor correspondiente.


Ver los documentos ClientVsServer (ClienteVsServidor), Server (Servidor), y ServerOptions (OpcionesDeSevidor) y Tutorial dentro del sistema de ayuda de SuperCollider para más información.
