Funciones y mensajes .scope


Una manera fácil de audicionar procesosz de síntesis es probarlos dentro de una función. Para hacer esto, agregue un mensaje de .scope o .play a la función. El mensaje .scope, que funciona únicamente con el sevidor interno, muestra una representación visual de la onda sonora (a la manera de un osciloscopio). 



////////////////////////////////////////////////////////////////////////////////////////////////////


Encender el servidor interno


Server.internal.boot;


Correr este ejemplo y ver la ventana del scope.


// probar un proceso de síntesis en una función

(

{ 

SinOsc.ar([440.067, 441.013], 0, 1) 

* 

SinOsc.ar([111, 109], 0, 0.2) 

}.scope;

)

Expansión multicanal


Expander un ugen a dos canales con un arreglo en cualquiera de los de los espacios de argumentos (controles).


{ Saw.ar([500, 933], 0.1) }.scope;


Otra manera (más larga) de escribir la misma cosa es


{ [ Saw.ar(500, 0.1), Saw.ar(933, 0.1)] }.scope;


Expandir un ugen a tres canales añadiendo values al arreglo.


{ Saw.ar([500, 933, 2033], 0.1) }.scope;


// 4 canales


{ Saw.ar([500, 933, 2033, 895], 0.1) }.scope;
