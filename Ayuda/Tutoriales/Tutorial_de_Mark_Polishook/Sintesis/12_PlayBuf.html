Separar los procesos de síntesis en partes que llevan a cabo tareas pequeñas y bien definidas fomenta el diseño modular y el reuso de componentes (el mantra de la programación orientada a objetos).


(

// lee un archivo de audio del disco duro

b = Buffer.read(s, "sounds/a11wlk01.wav");


// un reproductor de muestras en mono ... un canal solo

SynthDef("unReproductorMono", { arg bus = 0, bufnum = 0, rateScale = 1;

Out.ar(

bus,

PlayBuf.ar(

1,

bufnum,

BufRateScale.kr(bufnum) * rateScale

)

*

EnvGen.kr(Env.sine(BufDur.kr(bufnum)))

)

}).load(s);

)


(

// prueba el SynthDef ... ¿funciona? (sí, está bien. suena en el canal izquierdo)

Synth("unReproductorMono", [\bus, 0, \bufNum, b]);

)



(

// un ejemplo simple de reutilización de componentes ... usa el argumento \bus para asignar

// a distintos canales dos sintetizadores construidos a partir del mismo SynthDef 

// en este caso, reproduce un archivo de audio de un canal en dos canales

// una velocidad de reproducción distinta para cada canal hace que el efecto sea más obvio

Synth("unReproductorMono", [\bus, 0, \bufNum, b, \rateScale, 0.99]);

Synth("unReproductorMono", [\bus, 1, \bufNum, b, \rateScale, 1.01])

)

Información

 

Los UGens BufRateScale y BufDur, como fue mostrado en el ejemplo anterior, controlan la velocidad con la que PlayBuf reproduce el archivo de audio y la duración del envolvente aplicado al PlayBuf.


BufRateScale y BufDur son de una familia  de UGens que heredan de InfoUGenBase o de BufInfoUGenBase.


Para ver la lista completa de estos UGens, evalúa

Haz un "loop" a una muestra


El siguiente ejemplo usa tres SynthDefs para hacer una cadena. El primer SynthDef es un reproductor de muestras que hace "loop" a través de un buffer. El segundo SynthDef hace una modulación de anillo a su entrada. El tercer SynthDef aplica un filtro pasa bajas.


(

// lee un archivo de audio

b = Buffer.read(s, "sounds/a11wlk01.wav");


// define un reproductor de muestras va a hacer "loop" a un archivo de audio

SynthDef("unReproductorQueHaceLoop", { arg outBus = 0, bufnum = 0, rateScale = 1, mul = 1;

Out.ar(

outBus,

PlayBuf.ar(

1,

bufnum,

BufRateScale.kr(bufnum) * rateScale + LFNoise1.kr(2.reciprocal, 0.05),

loop: 1 // reproduce el archivo de audio una y otra vez sin parar

)

*

mul

)

}).load(s);


// aplica una modulación de amplitud a una fuente de audio

SynthDef("ampMod", { arg inBus = 0, outBus = 0, modFreq = 1;

Out.ar(

outBus,

[ // el UGen In.ar lee de un canal de audio

In.ar(inBus, 1) * SinOsc.kr(modFreq), 

In.ar(inBus, 1) * SinOsc.kr(modFreq - 0.02)

]

)

}).load(s);


// aplica un filtro pasa bajas a una fuente de audio

SynthDef("unFiltroPasaBajas", { arg inBus = 0, outBus = 0, freq = 300, freqDev = 50, boost = 1;

Out.ar(

outBus,

RLPF.ar(

In.ar(inBus, 2),

Lag.kr(LFNoise0.kr(1, freqDev, freq), 1),

0.2

)

*

boost 

* 

LFPulse.kr(1, [0.25, 0.75], [0.5, 0.45])

+

In.ar(inBus, 2)

)

}).load(s);

)


// define 2 grupos, 1 para el material fuente y el otro para los efectos

(

~fuente = Group.head(s);

~efecto = Group.tail(~s);

)


(

// añade el reproductor de archivos de audio al grupo fuente

~laFuente = Synth.head(

~fuente, 

"unReproductorQueHaceLoop", [\outBus, 3, \bufNum, b, \rateScale, 1, \mul, 0.051]);

// añade un sintetizador que haga modulación de amplitud en la cabeza del grupo de efectos

~fx1 = Synth.head(

~efecto,

"ampMod", [\inBus, 3, \outBus, 5, \modFreq, 1]);

// añade filtraje en la cola del grupo de efectos

~fx2 = Synth.tail(

~efecto,

"unFiltroPasaBajas", [\inBus, 5, \outBus, 0, \boost, 5])

)


// examina los nodos

(

s.queryAllNodes;

)

     /  \

~fuente  ~efectos // ~fuente y ~efectos son grupos

 |        |      \

 |        |       \

 synth    synth    synth

 


// Cambiar los valores de los argumentos (control) afecta al timbre

(

 ~laFuente.set(\rateScale, 0.95.rrand(1.05), \mul, 0.051.rrand(0.07));

 ~fx1.set(\modFreq, 800.0.rrand(1200));

 ~fx2.set(\freq, 500.rrand(700), \freqDev, 180.rrand(210), \boost, 7);

)
