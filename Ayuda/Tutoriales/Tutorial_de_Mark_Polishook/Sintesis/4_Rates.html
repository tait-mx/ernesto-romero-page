Audio rate


Los 'Ugens' para los que se utiliza el mensaje '.ar' corren a una frecuencia de muestreo de audio, es decir, a 44,100 muestras por segundo. Utiliza el mensaje '.ar' a los 'Ugens' cuando sean parte de  la cadena de audio que va a ser escuchada.


SinOsc.ar(440, 0, 1);


////////////////////////////////////////////////////////////////////////////////////////////////////


Control rate


Los 'Ugens' para los que se utiliza el mensaje '.kr' corren a una frecuencia de muestreo de control


SinOsc.kr(440, 0, 1);


Por defecto, los 'Ugens' que utilizan una frecuencia de muestreo de control generan un valor por cada sesenta y cuatro muestras generadas por un 'Ugen' de audio. Los 'Ugens' de Control Rate utilizan, por lo tanto, mucho menos recursos de la computadora que los 'Ugens' de audio.


Utiliza los 'Ugens' de Control Rate como moduladores, es decir, como señales que controlan una señal de audio.


////////////////////////////////////////////////////////////////////////////////////////////////////


Aquí, un SinOsc de Control Rate modula la frecuencia de una onda 'Pulse' que corre en Audio Rate:


(

SynthDef("anExample", {

Out.ar(

0,

Pulse.ar(

[220, 221.5] + SinOsc.kr([7, 8], 0, 7), // la Control Rate utiliza pocos ciclos de CPU

0.35,

0.02

)

)

}).load(s);

)


Synth("anExample")
