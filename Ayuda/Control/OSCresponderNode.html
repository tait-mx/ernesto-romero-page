OSCresponderNode respondedor de red del lado del cliente



Registra una función que será llamada al recibir un comando específico desde una dirección específica de OSC.

Misma interface que OSCresponder, pero permite respondedores múltiples para el mismo comando.


notar que OSCresponderNode evalúa su función en el proceso del sistema.

para poder accesar a proceso de la aplicación (e.g. para accesar a GUI ) usar { ... }.defer;




Definiendo un OSCresponder para escuchar a una aplicación remota


// ejemplo: dos aplicaciones SuperCollider comunicandose


// applicación 1:

n = NetAddr("127.0.0.1", 57120); // el url debe de ser el de la computadora con la aplicación 2 (o nil)


o = OSCresponder(n, '/chat', { |t, r, msg| ("tiempo:" + t).postln; msg[1].postln }).add;


// applicaciónn 2:

m = NetAddr("127.0.0.1", 57120); // el url debe de ser el de la computadora con la aplicación 1

m.sendMsg("/chat", "Hola aplicación 1");


// enviando paquetes (incluyendo marcas de tiempo (timestamps))

(

m.sendBundle(2.0, ["/chat", "Hola aplicación 1"], ["/chat", "Hola mundo"]);

m.sendBundle(0.0, ["/chat", "Hola aplicación 1"], ["/chat", "Hola mundo"]);

)


// finaliza applicaciónn 2:

m.disconnect;


// finaliza applicaciónn 1:

n.disconnect; o.remove;
